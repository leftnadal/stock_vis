> 관련 문서: stock-vis-preset-reference.md (전략 40개 + 이론 17개 상세)

## 1. 개요

### 1-1. 목적

Portfolio Coach의 프리셋 시스템은 사용자가 자신의 투자 철학에 맞는 이론적 렌즈를 선택하고, 그 관점에서 보유 종목을 분석받는 기능이다. 단순한 점수표가 아니라, "이 이론의 관점에서 내 포트폴리오는 어떤 상태인가"를 진단하고, 약한 부분을 발견하면 Screener → Chain Sight → Thesis Control로 이어지는 순환 루프를 통해 포트폴리오를 개선해 나가는 구조이다.

### 1-2. 핵심 원칙

- **이론 기반**: 40개 이상의 투자 전략과 17개 포트폴리오 구성 이론을 연구하여, 학술적·실무적으로 검증된 프레임워크만 프리셋으로 제공
- **관점의 다양성**: 같은 종목이라도 프리셋에 따라 다른 진단이 나옴. MSFT는 GARP에서는 "PEG 양호", 배당에서는 "배당성장 20년 연속", Buffett에서는 "moat 강함"으로 각각 다른 이야기
- **종목 추천 금지**: Coach는 "이 종목을 사라"는 비허용. Screener를 별도 도구로 제공하여 책임 분리
- **순환 루프**: Coach 진단 → Screener → Chain Sight → Thesis Control → Portfolio → Coach 재진단
- **락인 효과**: 사용자가 쓸수록 Chain Sight 관계 맵이 확장되어, 다른 서비스로 이동 시 재구축 비용 발생

## 2. 투 트랙 구조

### 2-1. 트랙 A: 실제 포트폴리오 (자산 관리)

사용자가 실제로 보유한 종목의 자산 현황을 관리하는 트랙.

| 항목 | 내용 |
|---|---|
| 목적 | 실제 자산 현황 파악, 추이 분석, 세금·수익률 관리 |
| 데이터 | 종목, 수량, 매수가, 현재가, 평가금액, 수익률, 배당 수령 내역 |
| 분석 | 자산 추이 그래프, 총 수익률, 일간 변동, 섹터/thesis 비중 |
| 프리셋 관여 | 없음. 순수 팩트 기반 |
| 관련 기능 | FR-P100(입력), FR-P200(현황), FR-P600(History), FR-P700(Review) |

### 2-2. 트랙 B: 분석 테이블 (관점별 렌즈)

사용자가 특정 투자 이론을 선택하여 테이블을 만들고, 종목을 넣어 해당 관점으로 분석하는 트랙.

| 항목 | 내용 |
|---|---|
| 목적 | 특정 투자 이론의 관점에서 종목을 분석·진단 |
| 데이터 | 해당 프리셋이 사용하는 지표 전체 + 시장 대비 백분위 점수 |
| 대상 종목 | 보유 종목, 예비 후보, 보유하지 않지만 궁금한 종목 모두 가능 |
| 종목 관리 | 복사(다른 테이블에서 같은 종목을 다른 관점으로 분석), 이동 가능 |
| 프리셋 관여 | 핵심. 테이블 = 프리셋. 테이블을 만들 때 프리셋을 선택 |
| 관련 기능 | FR-P500(Coach), FR-P800(Style), Screener |

### 2-3. 두 트랙의 연결

```
[트랙 A: 실제 포트폴리오]
    │
    ├─→ "내 보유 종목 전체를 GARP으로 분석" 
    │     → 트랙 B의 GARP 테이블에 보유 종목 자동 복사
    │
    ├─→ "내 보유 종목 전체를 배당 성장으로 분석"
    │     → 트랙 B의 배당 성장 테이블에 보유 종목 자동 복사
    │
[트랙 B: 분석 테이블]
    │
    ├─→ "이 종목을 예비 후보로 등록"
    │     → 트랙 A의 예비 후보 목록에 연결
    │
    ├─→ "이 종목을 Chain Sight에서 관계 탐색"
    │     → Chain Sight로 이동 (순환 루프 시작)
    │
    └─→ 사후 분석: "이 시점에 GARP 테이블의 진단이 이랬고, 
          그 이후 실제 수익률이 이랬다" (트랙 A 데이터와 교차 비교)
```

## 3. 프리셋 구조

### 3-1. 2단계 구조

```
카테고리 (8~9개)
  └── 세부 프리셋 (카테고리당 약 3개, 총 16~25개)
```

- 온보딩 시 카테고리만 선택해도 디폴트 프리셋 적용
- 관심 있는 사용자는 세부 프리셋까지 선택
- 여러 테이블을 만들어 여러 프리셋을 동시 사용 가능

### 3-2. 카테고리 목록

| # | 카테고리 | 설명 |
|---|---|---|
| 1 | 가치 투자 | 저평가 기업, 안전마진, 펀더멘털 중시 |
| 2 | 성장 투자 | 빠르게 성장하는 기업, 미래 가치 중시 |
| 3 | 배당/인컴 | 현금흐름, 배당 안정성·성장 중시 |
| 4 | 모멘텀/추세 | 가격 추세, 시장 흐름, 상대 강도 중시 |
| 5 | 퀄리티/방어 | 수익성, 경쟁우위, 재무 건전성, 하방 보호 중시 |
| 6 | 퀀트/팩터 | 여러 팩터 결합, 데이터 기반 체계적 접근 |
| 7 | 역발상 | 시장 과잉 반응에 반대 베팅, 저평가 기회 |
| 8 | 테마/전략 | 특정 메가트렌드나 투자 철학 중심 |
| 9 | 포트폴리오 구성 | 종목 조합·비중·리스크 배분 관점 (선택적) |

### 3-3. 세부 프리셋 선정 기준

각 카테고리에서 약 3개씩 선정. 선정 기준:

- **Stock-Vis 데이터로 구현 가능한가**: FMP 재무 데이터 + 가격 데이터 + Chain Sight 관계 데이터 + Thesis Control 데이터로 지표 계산 가능 여부
- **프리셋 간 차별화가 명확한가**: 같은 카테고리 내 프리셋끼리 지표 구성이 충분히 다른가
- **사용자 수요가 있는가**: 실제 투자자들이 사용하는 검증된 전략인가

### 3-4. 세부 프리셋 후보 (Step 1에서 최종 확정 예정)

**적합도 ★★★★★ (핵심 후보):**
- Buffett Quality Value, GARP, Quality Growth/Compounder

**적합도 ★★★★ (강력 후보):**
- Magic Formula, Piotroski F-Score, CANSLIM, Dividend Growth, Shareholder Yield, Quality Factor, Wide Moat, Multi-Factor

**적합도 ★★★ (조건부 후보):**
- Deep Value, Pure Growth, Dividend Aristocrats, High Yield Income, Price Momentum, Relative Strength, Low Volatility, Contrarian, Thematic Investing, Concentrated Portfolio 등

> 상세 목록 및 각 전략의 철학·지표·장단점·사용자 설명문은 `stock-vis-preset-reference.md` 참조

## 4. 점수 체계

### 4-1. 가중치 불필요

기존 설계에서 검토했던 "공통 지표 + 프리셋 전용 지표 + 가중치 매트릭스" 구조는 폐기.

**폐기 이유:**
- 테이블 구조(프리셋 = 테이블)에서는 해당 프리셋에 맞는 종목만 들어가므로, 모든 지표가 의미 있음
- 공통/전용 분리 불필요 — 프리셋이 사용하는 지표를 전부 보여주면 됨
- 가중치로 종합 점수를 만드는 것보다 개별 지표 점수 + LLM 코멘트가 더 유용

**대신 필요한 것:**
- 프리셋별 사용 지표 목록
- 각 지표의 적정 범위 (프리셋마다 다를 수 있음)
- 시장 대비 백분위 기반 점수

### 4-2. 시장 대비 백분위 점수

| 항목 | 내용 |
|---|---|
| 비교 기준 | 시장 전체 종목 대비 백분위 |
| 구간 | 5구간 |
| 1점 | 하위 0~20% (시장 대비 매우 약함) |
| 2점 | 하위 20~40% (시장 대비 약함) |
| 3점 | 중간 40~60% (시장 평균) |
| 4점 | 상위 20~40% (시장 대비 강함) |
| 5점 | 상위 0~20% (시장 대비 매우 강함) |

**주의:** 지표 성격에 따라 높은 것이 좋은 지표(ROE, EPS 성장률)와 낮은 것이 좋은 지표(P/E, 부채비율)가 있으므로, 점수 매핑 시 방향을 반영.
- ROE: 상위 20% = 5점 (높을수록 좋음)
- P/E: 하위 20% = 5점 (낮을수록 좋음, 단 프리셋에 따라 다를 수 있음)
- 부채비율: 하위 20% = 5점 (낮을수록 좋음)

### 4-3. 지표별 점수 집계

```
테이블 내 종목 5개, PEG 지표 예시:

종목A: 시장 상위 10% → 5점
종목B: 시장 상위 15% → 5점
종목C: 시장 상위 25% → 4점
종목D: 시장 상위 30% → 4점
종목E: 시장 상위 45% → 3점

PEG 지표 합산: 5+5+4+4+3 = 21점
5점 개수: 2개
```

### 4-4. 지표 정렬 순서

**1차 정렬:** 합산 점수 높은 순 → 낮은 순 (강점부터 약점 순서)

**타이브레이커 (합산 점수 동일 시):**
1. 5점 개수가 많은 지표 우선
2. 4점 개수가 많은 지표 우선
3. 3점 개수가 많은 지표 우선
4. 2점 개수가 많은 지표 우선
5. 1점 개수가 많은 지표 우선

**효과:**
- 사용자는 위에서부터 "내 포트폴리오가 시장 대비 어디가 강한지" 확인
- 아래로 갈수록 "시장 대비 어디가 약한지" 확인
- 같은 이론으로 골라서 점수가 비슷해지는 "평균의 함정" 방지 (시장 대비 상대 위치이므로 지표별 차이 발생)

### 4-5. 백분위 데이터 산출

| 항목 | 내용 |
|---|---|
| 데이터 소스 | FMP API |
| 대상 | S&P 500 또는 전체 상장 종목 (추후 결정) |
| 갱신 주기 | EOD 배치 (1일 1회) |
| 저장 | 각 지표별 백분위 테이블 사전 계산 |
| 사용 시점 | 분석 테이블에 종목 추가/갱신 시 백분위 조회 |

## 5. 진단 출력 구조

### 5-1. 지표별 진단

프리셋이 사용하는 모든 지표를 합산 점수 순으로 나열. 각 지표마다:

```
[지표명] — [합산 점수] / [만점]
🟢 5점: 종목A, 종목B (시장 상위 20% 이내)
🟢 4점: 종목C, 종목D
🟡 3점: 종목E
🔴 2점: (해당 없음)
🔴 1점: (해당 없음)
[LLM 한 줄 코멘트]
```

**예시 (GARP 테이블, 5개 종목):**

```
ROIC — 24/25
🟢 5점: NVDA, MSFT, AMZN, LLY (4개)
🟢 4점: META (1개)
"전 종목이 시장 상위 40% 이내. 자본 효율성이 매우 우수합니다."

EPS 성장률 — 22/25
🟢 5점: NVDA, AMZN, META (3개)
🟢 4점: MSFT (1개)
🟡 3점: LLY (1개)
"대부분 시장 상위권 성장률. LLY는 업종 특성상 성장 속도가 상대적으로 완만합니다."

PEG — 19/25
🟢 5점: LLY (1개)
🟢 4점: AMZN, MSFT (2개)
🟡 3점: META (1개)
🔴 2점: NVDA (1개)
"NVDA의 성장 대비 밸류에이션이 시장 평균 이하. PEG 관점에서 재검토 필요."

부채비율 — 15/25
🟢 4점: LLY (1개)
🟡 3점: NVDA, MSFT (2개)
🔴 2점: AMZN (1개)
🔴 1점: META (1개)
"META와 AMZN의 부채 수준이 시장 대비 높은 편. GARP 관점에서 주의 필요."
```

### 5-2. 진단 카드 (상위 3개 문제점)

합산 점수가 가장 낮은 지표 3개를 기반으로 LLM이 생성.

```
🟡 카드 1: 부채비율이 시장 대비 높습니다
  META(D/E 1.4)와 AMZN(D/E 1.2)이 시장 하위 40%에 위치합니다.
  GARP 관점에서 부채가 낮을수록 성장의 지속성이 높아집니다.
  
🟡 카드 2: NVDA의 PEG가 적정 범위를 초과합니다
  성장률은 우수하지만 밸류에이션이 이를 상쇄하고 있습니다.
  시장 대비 PEG 하위 40%에 위치.

🟡 카드 3: thesis 미설정 종목 2개
  LLY, META에 투자 근거가 설정되어 있지 않습니다.
  thesis를 세우면 더 정확한 분석이 가능합니다.
```

### 5-3. 종합 점수

모든 지표의 합산 점수를 총합하여 100점 만점으로 환산.
- 메인 화면에서 한눈에 현재 상태 파악
- 시간에 따른 추이 추적 (지난달 72점 → 이번 달 78점)
- 프리셋 변경 전후 비교

## 6. 순환 루프

### 6-1. 전체 플로우

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  [Portfolio Coach 진단]                                  │
│    분석 테이블에서 약한 지표 발견                          │
│    "부채비율이 높고, 배당이 없는 종목이 많습니다"           │
│         │                                               │
│         ▼                                               │
│  [Screener]                                             │
│    프리셋 기준으로 후보 종목 필터링                        │
│    "GARP 기준 + 부채비율 낮은 + 배당 있는 종목 검색"       │
│         │                                               │
│         ▼                                               │
│  [Chain Sight]                                          │
│    후보 종목의 관계 탐색                                  │
│    기존 보유 종목과의 관계(공급망, 경쟁, 협력) 확인         │
│    새로운 관계 노드 추가 → 사용자 관계 맵 확장             │
│         │                                               │
│         ▼                                               │
│  [Node Monitoring → 1차 검증 → Thesis Control]           │
│    후보 종목에 대한 투자 가설 설정                         │
│    전제 유효성 확인                                       │
│         │                                               │
│         ▼                                               │
│  [Portfolio 예비 후보 등록]                               │
│    분석 테이블에 추가 → 기존 종목과 함께 재진단             │
│    "이 종목을 추가하면 부채비율 지표가 개선됩니다"           │
│         │                                               │
│         ▼                                               │
│  [Coach 재진단] ──────────────────────────→ (루프 반복)    │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 6-2. 역할 분리

| 구성요소 | 역할 | 허용 범위 |
|---|---|---|
| Coach | 포트폴리오 구조 진단 | 지표별 강약점 분석, 문제 지적 |
| Coach | 비허용 | "이 종목을 사라" 류의 직접 추천 |
| Screener | 종목 필터링 도구 | 프리셋 기준에 맞는 종목 목록 제공 |
| Screener | 책임 구조 | 사용자 주도 (Coach 추천이 아님) |
| Chain Sight | 관계 탐색 | 종목 간 숨겨진 관계 발견 |
| Chain Sight | 락인 역할 | 사용할수록 관계 맵 확장 → 이탈 비용 증가 |

## 7. 프리셋 이론적 기반

### 7-1. 종목 선택 전략 (40개, 8개 카테고리)

각 카테고리에서 약 3개씩 Stock-Vis 적합도 기준으로 선정 예정.

| 카테고리 | 전략 수 | 핵심 후보 (★★★★ 이상) |
|---|---|---|
| 가치 투자 | 6개 | Buffett Quality Value, Magic Formula, Piotroski F-Score |
| 성장 투자 | 6개 | GARP, CANSLIM, Quality Growth/Compounder |
| 배당/인컴 | 4개 | Dividend Growth, Shareholder Yield |
| 모멘텀/추세 | 4개 | Price Momentum, Relative Strength |
| 퀄리티/방어 | 5개 | Quality Factor, Wide Moat |
| 퀀트/팩터 | 4개 | Multi-Factor |
| 역발상 | 4개 | Contrarian |
| 테마/기타 | 7개 | Thematic Investing, Concentrated Portfolio |

> 각 전략의 상세 내용(철학, 지표, 장단점, 사용자 설명문)은 `stock-vis-preset-reference.md` 참조

### 7-2. 포트폴리오 구성 이론 (17개)

프리셋 진단에 이론적 근거를 제공하는 포트폴리오 이론.

**1순위 — MVP 반드시 반영:**

| 이론 | 적용 영역 |
|---|---|
| MPT (현대 포트폴리오 이론) | 상관관계/집중도 분석 — 모든 프리셋 기본 인프라 |
| PMPT (포스트 모던 포트폴리오 이론) | 하방 리스크 — Sortino Ratio 사용 |
| Risk Parity (리스크 패리티) | 리스크 기여도 분석 — "비중 15%지만 리스크 35%" |

**2순위 — MVP 부분 반영, Phase 2 강화:**

| 이론 | 적용 영역 |
|---|---|
| Black-Litterman | Thesis 확신도 기반 비중 적정성 — Thesis Control 직접 연결 |
| HRP (계층적 리스크 패리티) | 클러스터 기반 분산 — Chain Sight 관계 그래프 직접 연결 |
| Kelly Criterion | 확신도 기반 비중 — Concentrated Portfolio 진단 |
| BPT (행동 포트폴리오 이론) | 행동 편향 분석 — 처분 효과, 조기 익절 등 패턴 감지 |

**3순위 — Phase 3 이후:**

| 이론 | 적용 영역 |
|---|---|
| Fama-French | 프리셋별 팩터 노출도 분석 |
| Resampled Efficiency | 시뮬레이션 기반 고급 진단 |
| Robust Optimization | Thesis 확신도 → 불확실성 범위 변환 |

> 각 이론의 상세 내용은 `stock-vis-preset-reference.md` 참조

## 9. 남은 작업

### Step 1: 카테고리별 프리셋 3개씩 추리기
- 8~9개 카테고리 × 약 3개 = 24~27개 → 최종 16~25개로 조정
- 선정 기준: Stock-Vis 구현 가능성, 프리셋 간 차별화, 사용자 수요

### Step 2: 프리셋별 사용 지표 전체 목록 확정
- 각 프리셋이 사용하는 지표를 빠짐없이 나열
- 지표별 적정 범위 정의 (프리셋마다 다를 수 있음)
- 시장 대비 백분위 방향 정의 (높을수록 좋은 지표 vs 낮을수록 좋은 지표)

### Step 3: 테이블 구조 설계
- 트랙 A(실제 포트폴리오)와 트랙 B(분석 테이블) 간 데이터 연결
- 종목 복사/이동 UX
- 분석 테이블 생성/삭제/관리 UX

### Step 4: LLM 프롬프트 설계
- 지표별 한 줄 코멘트 생성용 프롬프트
- 진단 카드(상위 3개 문제점) 생성용 프롬프트
- 프리셋별 톤/관점 차이 반영

### Step 5: DB 모델 설계
- 분석 테이블 모델 (AnalysisTable)
- 프리셋-지표 매핑 모델
- 백분위 캐시 테이블
- 진단 결과 저장 모델

### Step 6: API 설계
- 분석 테이블 CRUD
- 진단 요청/결과 조회
- 백분위 배치 업데이트

### Step 7: UI 와이어프레임
- 트랙 A / 트랙 B 전환 UX
- 분석 테이블 메인 화면 레이아웃
- 지표별 진단 카드 레이아웃
