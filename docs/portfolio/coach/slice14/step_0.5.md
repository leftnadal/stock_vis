[슬라이스 14 / Step 0.5] 작업 지시서
게이트 가치 검증 probe — gate_tiers를 calibration할 가치가 있는가
═══════════════════════════════════════════════════════════════

■ 이 probe가 존재하는 이유 (맥락)

- Slice 13에서 ADDITIVE로 붙인 3단 게이트(gate_tiers / \_evaluate_gate_tier)는
  현재 12개 preset 전부 None — 100% 휴면 상태다. 시스템은 게이트 없이 정상
  동작 중.
- 게이트의 가치 명제 = "도메인 절대 기준의 위험을, 점수(상대 순위)와
  LLM 진단 코멘트가 놓치는 부분까지 결정론적으로 잡아준다."
- 그러나 이 명제는 검증된 적이 없다. 진단 코멘트를 쓰는 LLM이 이미 원시
  지표를 보고 절대 위험을 스스로 짚는다면, 게이트는 LLM과 겹치는 중복
  장치이고 calibration 슬라이스를 들일 가치가 없다.
- 본 probe는 그 명제를 $0.05~0.12짜리 작은 실험으로 검증한다. 결정 규칙은
  아래에 사전 등록돼 있다. probe 결과 → build(Part 1 진행) 또는 보류로 분기.
- 이 probe는 production 코드를 일절 변경하지 않는다. LLM 호출만 한다.

■ 사전 조건 (반드시 먼저 수행)

- 브랜치 slice14, HEAD = 63fedaf (#63 cost ledger, working tree clean).

■ 작업 1 — 케이스 선정 및 동결 (생성 이전에 커밋)
목적: "게이트 calibration 없이도 명백히 위험한" 케이스 8건을 고정한다.
───────────────────────────────────────────────────────────────
[케이스 선정 기준 — 8건 전부 충족할 것]
(a) 1 케이스 = 1 preset + 그 preset의 Core 지표 1개.
(b) 그 Core 지표가 preset의 정의적 전제를 정면으로 깨뜨리는 극단값.
"calibration된 경계값 없이도, 누가 봐도 위험"이라 말할 수 있어야 함.
예시(형태 예시일 뿐, 실제 preset/지표명은 코드 기준): - income(배당) preset인데 배당수익률이 0%에 가까움 - growth preset인데 이익 성장률이 음수 - value preset인데 밸류에이션 배수가 극단적으로 높음(=비쌈)
(c) 그 위험을 1문장으로 명확히 진술할 수 있을 것.
(d) 8건이 5개 카테고리(value/growth/income/factor/special) 중 최소
4개를 커버.
(e) 한 케이스 안의 나머지 지표는 대체로 정상/중립 범위 — 위험 지표를
1개로 격리해, "LLM이 그 1개를 짚는가"가 깨끗이 측정되도록 함.
(f) 데이터 출처: 기존 fixture(sample_analysis_context.py 등)에 (b)를
만족하는 극단이 있으면 그것을 쓰고, 없으면 최소한의 합성 입력
컨텍스트를 구성(구조는 현실적, 위험 지표만 극단값). 어느 쪽이든
케이스별로 출처를 명시.

■ 작업 3 — 평가 및 집계
───────────────────────────────────────────────────────────────
[평가 루브릭 — 사전 등록, 생성당 이진 판정]
각 생성 출력에 대해 "위험 자가 포착" 여부를 이진 판정한다.

- 포착(1): 코멘트가 해당 위험 지표를 우려/약점/리스크로 명시하고,
  경고적 framing(주의를 환기하는 톤)을 동반.
- 미포착(0): 지표 수치를 중립적으로 언급만 하거나 아예 언급 안 함.
  경고 framing이 없으면 미포착으로 본다(보수적 판정 — "LLM이 능동적
  으로 경고하는가"를 묻는 것이므로).
- 경계 사례는 0으로 처리하고, 그 사실을 평가 문서에 표시.

[집계]

- 케이스별: 3회 중 포착 횟수(0/3 ~ 3/3).
- 전체: 24 생성 중 포착 합계 → 포착률(%).
- 비결정성 지표: "케이스 내 결과가 갈린"(1/3 또는 2/3) 케이스 수.
  이 숫자가 게이트의 결정론적 일관성 가치를 판단하는 직접 근거.

───────────────────────────────────────────────────────────────
■ 사전 등록된 결정 규칙 (probe 실행 전 고정 — 변경 금지)
───────────────────────────────────────────────────────────────
포착률(24 생성 기준)에 따라:

- ≥ 75% (≥18/24) → LLM이 위험을 대체로 자가 포착함. 게이트는 LLM과
  중복 → calibration 보류 권고. 로드맵 "preset threshold" 항목은
  "LLM 커버로 의식적 종결"로 기록. gate_tiers는 휴면 유지(제거 안 함).
- ≤ 50% (≤12/24) → LLM이 위험을 자주 놓침. 게이트가 명백한 추가
  가치 → Part 1(게이트 구축) 진행 권고.
- 50~75% (13~17/24) → 판단 구간. 게이트의 결정론적 일관성(100%)이
  LLM의 부분 커버 + 비결정성 대비 한 슬라이스 값을 하는지 판단.
  비결정성 케이스 수가 클수록 게이트 쪽 근거가 강해짐.
  Claude Code는 이 규칙에 따른 잠정 결론까지만 제시한다. build/보류
  최종 확정은 병진이 평가 문서를 리뷰한 뒤 내린다.
