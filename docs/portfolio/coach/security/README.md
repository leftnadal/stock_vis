## 목적

Stock-Vis 프로젝트의 **보안 작업(취약점 스캔 · 패치 · 마이그레이션)을
체계적으로 누적**할 수 있는 단일 기록처. 일반 슬라이스가
`docs/portfolio/coach/slice{N}/`에 기록되는 것과 동일한 컨벤션을
따르되, "보안" 도메인 작업만 별도 트랙으로 분리해 추적성을 높인다.

C 계획(C-1 프론트 hotfix / C-2 백엔드 의존성 / C-3 Django 5.2 LTS) 및
향후 모든 보안 작업이 여기에 기록된다.

---

## 폴더 구조

```
docs/portfolio/coach/security/
├── README.md            ← 본 문서: 기록 시스템 안내·규칙
├── SECURITY_LEDGER.md   ← 마스터 인덱스 (전체 보안 작업 대장)
├── advisories/          ← 취약점 스캔 결과 원본 보존
│   └── YYYY-MM_주제.md
└── slices/              ← 보안 슬라이스별 작업 기록
    └── {작업명}.md
```

| 경로 | 성격 | 단위 |
|------|------|------|
| `SECURITY_LEDGER.md` | 마스터 인덱스 | 모든 보안 작업을 한 줄로 추적 |
| `advisories/` | 스캔 원본 | 시점·도구별 취약점 스냅샷 (수정 X, 보존만) |
| `slices/` | 작업 기록 | 슬라이스별 closing 보고서 |

## 새 보안 작업 시 기록 규칙

### 새 스캔이 추가될 때
1. `advisories/YYYY-MM_주제.md` 신규 파일 작성
   - 예: `advisories/2026-05_dependency-audit.md`, `advisories/2026-07_python-cve-recheck.md`
2. 스캔 일자 / 사용 도구 / 핵심 발견을 상단에 명시
3. **원본 수치 보존** (이후 시점 비교용) — 다른 문서에 그대로 인용 가능

### 새 보안 슬라이스가 진행될 때
1. 분기 직후 `slices/{작업명}.md` 신규 작성 (진행 중에는 ▶ 상태)
2. closing 시점에 게이트 결과·커밋 해시·비용까지 보강
3. **`SECURITY_LEDGER.md`의 마스터 표에 1행 추가**
4. 다른 슬라이스에서 참조되면 `slices/c1_frontend_hotfix.md` 형태로 상대경로 링크

### 미해결 보안 부채가 생길 때
- `SECURITY_LEDGER.md`의 "미해결 보안 부채" 소절에 추가
- ID·요약·발생 시점·왜 미적용인지·해소 조건(자연 / 별도 슬라이스)을 함께

## 명명 규칙

| 대상 | 패턴 | 예시 |
|------|------|------|
| 스캔 보고 | `YYYY-MM_주제.md` | `2026-05_dependency-audit.md` |
| 슬라이스 기록 | `{시리즈ID}_{설명}.md` (snake_case) | `c1_frontend_hotfix.md` |
| 마스터 행 ID | C-1, C-2, ... 또는 도메인 약어 + 번호 | C-1, C-2, ML-1 |

## 상태 표기 범례

| 표기 | 의미 | LEDGER에서의 사용 |
|------|------|-------------------|
| **▶** | 진행 중 / 예정 | base·브랜치 확정 후 closing 전 |
| **✅** | 종결 | closing 보고 + 커밋 해시 명시 후 |
| **⏸** | 보류 / 대기 | 의존성 / 의도된 대기 / 후순위 |
| **❌** | 취소 / 폐기 | 진행 후 백아웃 또는 폐기 결정 |

## 일반 슬라이스 트랙과의 관계

- 보안 작업이라도 회귀·IDENTICAL·KPI 같은 일반 슬라이스 게이트는
  그대로 적용된다. C-1의 회귀 0 / vitest 65/65가 그 예시.
- `kpi_matrix.md`(전체 트랙) 갱신은 보안 슬라이스가 일반 KPI 베이스라인을
  변경할 때만 (예: 백엔드 회귀 카운트 변화) — C-1처럼 frontend/ 한정이면
  본 폴더 안에서만 기록.
- 보안 슬라이스도 `debts.md`와 동기화: 신규 부채는 양쪽에 등록 가능 —
  단 **이 폴더의 "미해결 보안 부채" 소절이 보안 도메인의 1차 소스**.
