> **작성 기준**: 현재 프로젝트 프론트엔드 패턴 (`Header.tsx`, `MarketOverview.tsx`, `MoverCard.tsx`) 참조

## 총평

설계 원칙("시그널이 주, 뉴스가 종")과 Chain Sight 철학이 잘 정렬되어 있다.
"숫자가 아니라 스토리"라는 핵심 원칙은 Invisible Education과 완벽하게 맞닿아 있다.
다만 **정보 계층 구조, 시그널 카드의 인지 부하, 모바일 레이아웃, Cold-start UX**에서
수정/보완이 필요한 구체적인 지점이 있다.

## 1. 정보 계층 구조 (Information Hierarchy)

### 현재 계획의 구조

```
[시장 요약 바] → [시그널 카드 그리드] → [뉴스 맥락 섹션] → [Chain Sight 미니맵]
```

### 문제점: 시장 요약 바의 정보 밀도

현재 시장 요약 바에 담긴 정보:

```
S&P 500  +0.82%   NASDAQ  +1.24%   VIX  14.2 (-5%)
오늘 시그널: 147건 / 847종목에서 감지
```

이 두 줄은 성격이 완전히 다른 정보다.

- 첫 줄: "시장이 어떤 날이었는가" (외부 컨텍스트)
- 둘째 줄: "이 앱이 오늘 얼마나 많이 발견했는가" (앱 내부 가치 증명)

사용자가 처음 앱을 열었을 때 가장 먼저 봐야 할 것은 "오늘 이 앱을 열 가치가 있는가"다.
두 정보를 같은 바에 혼재시키면 눈의 동선이 나뉘고, 어느 것도 집중해서 읽지 않게 된다.

**권장 계층 재설계:**

```
Level 1 — 오늘의 앱 가치 (한 줄 헤드라인, 가장 크게)
  "오늘 847종목에서 147개 시그널 감지 — 2026년 2월 26일 장 마감 기준"

Level 2 — 시장 맥락 (작고 조용하게, 보조 역할)
  S&P +0.82%  NQ +1.24%  VIX 14.2

Level 3 — 시그널 카드
  (아래로)
```

이렇게 하면 사용자는 "오늘 뭔가 있구나" → "시장은 어떤 날이었나" → "어떤 시그널이 있나"
순서로 자연스럽게 시선이 이동한다.

### 문제점: 뉴스 맥락 섹션의 위치

현재 계획에서 "데이터가 포착한 맥락" 섹션은 시그널 카드 아래에 위치한다.
그러나 이 섹션은 시그널 카드와 동등한 레이어가 아니다.
뉴스 맥락은 특정 시그널에 연결된 부속 정보다.

**권장**: 독립 섹션으로 두지 말고, 각 시그널 카드에 뉴스 힌트를 인라인으로 녹여넣어라.
카드 하단에 단 한 줄의 뉴스 헤드라인이 카드 맥락을 설명하게 하면,
시그널 카드 → 뉴스 연결이 자연스러운 흐름이 된다.

### 권장 최종 정보 계층

```
┌─────────────────────────────────────────────────────┐
│ [헤드라인] 오늘 847종목에서 147개 시그널 감지        │  ← Level 1: 앱 가치
│ [서브] S&P +0.82%  NQ +1.24%  VIX 14.2  [날짜]     │  ← Level 2: 시장 컨텍스트
├─────────────────────────────────────────────────────┤
│ [시그널 카테고리 필터 탭]                            │  ← Level 3: 탐색 진입
│  전체  모멘텀  거래량  돌파  반전  관계  기술지표    │
├─────────────────────────────────────────────────────┤
│ [시그널 카드 그리드]                                 │  ← Level 4: 핵심 콘텐츠
│  (각 카드에 뉴스 힌트 한 줄 인라인)                  │
└─────────────────────────────────────────────────────┘
```

## 2. 화면 레이아웃 검토

### 2-1. 메인 대시보드: 시그널 카드 그리드

현재 계획의 카드:

```
┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐
│💥 거래량  │ │🚀 급등주  │ │🕵️ 매집   │ │🐢 섹터   │
│  폭발    │ │          │ │  의심    │ │  소외주   │
│  23종목  │ │  12종목  │ │  5종목   │ │  7종목   │
│ NVDA+5.2%│ │ MSTR+8.1%│ │ XYZ ??? │ │ AMD -0.2%│
│ PLTR+3.1%│ │ COIN+6.3%│ │ ABC ??? │ │ INTC+0.1%│
└──────────┘ └──────────┘ └──────────┘ └──────────┘
← 가로 스크롤 →
```

**문제점 1: 2행 가로 스크롤의 인지 부하**

현재 8개 카드를 4x2 가로 스크롤로 배치했다.
가로 스크롤은 사용자에게 "여기에 더 있다"를 능동적으로 발견하게 요구한다.
2행 가로 스크롤은 특히 모바일에서 스크롤 방향 혼란(세로 vs 가로)을 유발한다.

**권장 레이아웃 전략:**

```
데스크톱 (>= 1280px): 4열 고정 그리드, 2행 = 8카드 모두 노출 (스크롤 불필요)
태블릿 (768px-1280px): 3열 그리드, 첫 행 3카드 + 나머지 가로 스크롤 힌트
모바일 (< 768px):      2열 그리드, 세로 스크롤로 모두 탐색
```

모바일에서 가로 스크롤 완전 제거. 모든 카드를 2x4 세로 그리드로 쌓아라.

**문제점 2: 카드 내 종목 리스트 (2개) 제한**

각 카드에 대표 종목 2개만 노출하는데, 이것만으로는 카드를 클릭할 동기가 약하다.

**권장**: 카드에 아래 요소를 추가해 클릭 동기를 강화하라:

```
┌────────────────────────────┐
│ 거래량 폭발          23종목 │  ← 타이틀 + 카운트
│ 평소의 3배 이상 거래         │  ← 서브타이틀 (자연어)
│                            │
│ NVDA  +5.2%  거래량 3.8x   │  ← 대표 종목 1
│ PLTR  +3.1%  거래량 4.2x   │  ← 대표 종목 2
│ + 21종목 더 보기           │  ← 나머지 종목 수 명시
│                            │
│ "Blackwell Ultra 양산 개시" │  ← 뉴스 힌트 1줄 (최신 것)
└────────────────────────────┘
```

"+ 21종목 더 보기" 텍스트 하나가 클릭 전환율을 크게 높인다.
사용자는 "이게 전부가 아니구나"를 느껴야 더 깊이 들어간다.

### 2-2. 시그널 카드 상세 화면

현재 계획에서 상세 화면의 구성:

```
← 뒤로  |  "이 시그널은?" 교육 섹션  |  정렬 컨트롤  |  종목 리스트
```

**문제점: 교육 섹션이 콘텐츠보다 먼저 나온다**

"이 시그널은?" 교육 박스가 종목 리스트보다 위에 위치한다.
초보 투자자가 처음 방문했을 때는 유용하지만,
두 번째, 세 번째 방문부터는 스킵하고 싶어지는 요소다.
이것이 Invisible Education 원칙에 위배된다.

**권장 재설계:**

교육 내용을 위에 강제 노출하는 대신, 카드 상단 타이틀 옆에
작은 물음표 아이콘(?)을 두고 탭 시 인라인 토글 확장으로 처리하라.

```
거래량 폭발 — 23종목  [?]     ← [?] 탭하면 교육 내용 펼침
────────────────────────────
정렬: 거래량비율  수익률  시총
────────────────────────────
[종목 리스트 바로 시작]
```

이렇게 하면 관심 있는 사람은 [?]를 탭해 배우고,
이미 아는 사람은 콘텐츠로 바로 진입한다.

**추가 권장: 종목 행의 정보 밀도 조정**

현재 계획의 종목 행:

```
MSTR   MicroStrategy    $412.80  +8.1%
Vol 5.1x  ███████████████████░░  Tech
미니차트
"비트코인 $100K 돌파에 따른 관련주 급등"
+골든크로스 +RSI과매수주의
[Chain Sight] [모니터링+]
```

이 6개 요소가 한 행에 쌓이면 카드 하나의 높이가 매우 커진다.
종목 20개가 있으면 스크롤이 극단적으로 길어진다.

**권장: 기본 행 + 확장 패턴**

```
기본 행 (항상 노출):
┌────────────────────────────────────────────────┐
│ MSTR  MicroStrategy  $412.80  +8.1%  Vol 5.1x  │
│ ████████████████████░░  [미니차트 spark]         │
└────────────────────────────────────────────────┘

탭/클릭 시 인라인 확장:
┌────────────────────────────────────────────────┐
│ "비트코인 $100K 돌파에 따른 관련주 급등"         │
│ +골든크로스  +RSI과매수주의                      │
│ [Chain Sight 보기]  [모니터링 추가]              │
└────────────────────────────────────────────────┘
```

이렇게 하면 한 화면에 더 많은 종목을 보여주면서,
관심 있는 종목만 확장해 깊은 정보를 볼 수 있다.

### 2-3. 종목 시그널 히스토리 화면

현재 설계는 적절하다. 아래 두 가지만 보완 권장.

**보완 1: 시그널 타임라인의 시각적 밀도 문제**

```
2/26  💥 거래량 3.8x  📈 골든크로스  🔥 3일 연속 상승
2/25  🔥 2일 연속 상승
2/24  (시그널 없음)
```

"시그널 없음" 행을 모두 표시하면 30일치 표시 시 빈 행이 지배적이 된다.
시그널 없는 날은 작은 점(.)으로 축약하거나 아예 생략하고
"시그널 있는 날만" 타임라인으로 표시하되, 날짜 간격으로 공백을 표현하라.

**보완 2: 기술적 지표 섹션의 맥락 부재**

```
RSI(14): 68.5   SMA50: $131.20   SMA200: $118.45
```

초보 투자자에게 이 숫자는 의미 없다. "숫자가 아니라 스토리" 원칙에 따라:

```
RSI 68.5  — 과열 구간 접근 중 (70 이상이면 주의)
SMA50 $131.20  — 현재가 $142.30은 50일 평균보다 8.5% 위
SMA200 $118.45 — 장기 상승 추세 유지 중
```

숫자 옆에 괄호로 맥락 한 줄을 추가하는 것만으로 Invisible Education이 완성된다.

### 2-4. Cold-start 화면

계획서에 Cold-start 플로우가 별도 섹션으로 정의되어 있으나,
Cold-start 화면 자체의 UI 스펙이 없다. 이것이 가장 중요한 화면이다.

**Cold-start 상황:**
- 신규 가입 사용자 (포트폴리오, 관심 종목 없음)
- 앱 첫 실행 시 "당신을 위한 대시보드"가 없는 상태

**현재 계획의 접근:** EOD 대시보드는 개인화와 무관하므로 Cold-start 문제가 없다
(모든 사용자에게 동일한 시그널 카드를 보여줌).

이 점은 장점이자 단점이다.

장점: 첫 방문부터 바로 가치 있는 콘텐츠를 보여줄 수 있다.
단점: "나를 위한 화면"이 아니라 "모두를 위한 화면"이라는 느낌.

**권장 Cold-start 온보딩 처리:**

신규 사용자에게는 시그널 카드 상단에 단 하나의 배너를 추가하라:

```
┌─────────────────────────────────────────────────┐
│  처음 오셨군요. 관심 섹터를 선택하면              │
│  맞춤 시그널을 먼저 보여드릴게요.                │
│  [반도체] [AI] [바이오] [에너지] [금융]  → 건너뜀 │
└─────────────────────────────────────────────────┘
```

이 선택을 저장해 시그널 카드 정렬에 반영하면,
최소한의 개인화로 "내 것"이라는 느낌을 줄 수 있다.
"건너뜀"을 항상 제공해 강요하지 않아야 한다.

## 3. 인터랙션 설계 검토

### 3-1. 카드 클릭 흐름

현재 계획: 시그널 카드 클릭 → 별도 상세 페이지로 이동.

**문제점**: 페이지 전환은 맥락 단절을 유발한다.
사용자가 메인 대시보드에서 카드를 클릭했을 때 돌아오기 버튼을 눌러야 한다는 것은
탐색 비용이 높다는 인식을 준다.

**권장: 슬라이드 오버 패널 패턴**

```
메인 대시보드 (유지)
       │
       ▼ 카드 클릭
┌──────────────────────────┐
│ 메인 대시보드 (흐릿하게) │  ← 완전히 사라지지 않음
│          ┌───────────────┤
│          │ 시그널 상세   │  ← 우측에서 슬라이드
│          │ 종목 리스트   │
│          │               │
│          └───────────────┤
└──────────────────────────┘
```

Next.js App Router에서 `sheet` 패턴 (shadcn/ui의 Sheet 컴포넌트)으로 구현 가능하다.
뒤로가기 버튼 없이 배경 탭이나 ESC로 닫는 방식이 Low-Action 원칙에 맞다.

모바일에서는 바텀 시트(bottom sheet)로 전환. 화면 하단에서 80% 높이로 올라오는 패턴.

### 3-2. 필터링 인터랙션

현재 계획에서 정렬 옵션:
```
정렬: [거래량 비율순 ▼] [수익률순] [시총순]
```

이것은 정렬만 있고 필터가 없다. 23종목 중 특정 섹터(예: 반도체)만 보고 싶은 경우,
또는 시총 $10B 이상만 보고 싶은 경우의 흐름이 없다.

**권장 필터 추가:**

```
[정렬: 거래량▼]  [섹터: 전체▼]  [시총: 전체▼]
```

단, 필터는 기본값이 "전체"여야 하고, 활성화된 필터만 강조 표시한다.
필터 패널이 팝업으로 열리는 것보다 인라인 드롭다운이 Low-Action에 맞다.

### 3-3. "모니터링+" 인터랙션

현재 계획: `[Chain Sight 🔗] [모니터링+]` 두 버튼이 종목 행마다 있다.

**문제점**: 두 버튼이 항상 노출되면 시각적 노이즈가 크다.
23종목이 있으면 46개의 버튼이 화면을 채운다.

**권장: Progressive Disclosure 패턴**

종목 행에는 버튼을 기본적으로 숨기고,
호버(데스크톱) 또는 탭(모바일) 시에만 액션 버튼을 노출한다.

```
기본 상태:
NVDA  $142.30  +5.2%  Vol 3.8x  ──────────────────

호버/탭 상태:
NVDA  $142.30  +5.2%  Vol 3.8x  [Chain Sight] [+추적]
```

이렇게 하면 리스트는 깔끔하게 유지되면서, 원하는 종목에서만 액션이 가능하다.

### 3-4. Chain Sight 전환 흐름

현재 계획: `[Chain Sight 🔗]` 클릭 → Chain Sight 그래프 페이지로 이동.

이 페이지 전환은 불가피하지만, 전환 후 "어디서 왔는지" 컨텍스트가 유지되어야 한다.

**권장**: Chain Sight 진입 시 `?signal=volume_spike&from=NVDA` 쿼리 파라미터를 유지하고,
Chain Sight 페이지 상단에 진입 컨텍스트 브레드크럼을 표시하라:

```
대시보드 > 거래량 폭발 > NVDA 관계 탐색
```

이것이 Zero Friction Navigation의 핵심이다.
사용자는 자신이 어디서 왔는지 알아야 안심하고 깊이 탐색한다.

## 5. Chain Sight DNA 기반 연쇄 발견 UX

### 5-1. 현재 계획의 Chain Sight 연결

계획서의 흐름:
```
메인 대시보드 → 시그널 카드 클릭 → 종목 클릭 → [Chain Sight] 버튼 → Chain Sight 그래프
```

이것은 3단계 전환이다. 각 단계마다 페이지/패널 이동이 발생하면
Chain Sight까지 도달하는 사용자가 극소수가 된다.

계획서의 관리자 분석 데이터에서 이미 예측하고 있다:
> "Chain Sight 전환율 (카드 → 종목 → CS): 섹터 소외주 42%, 급등주 18%"

급등주 카드에서 Chain Sight까지 도달하는 사용자가 18%라는 것은,
나머지 82%는 Chain Sight의 가치를 경험하지 못한다는 의미다.

### 5-2. Chain Sight 진입 마찰 감소 방안

**방안 A: 카드 레벨에서 Chain Sight 힌트 제공**

시그널 카드 자체에 관계 힌트를 넣어라.

```
┌────────────────────────────────┐
│ 거래량 폭발            23종목  │
│ 평소의 3배 이상 거래            │
│                                │
│ NVDA  +5.2%                    │
│ PLTR  +3.1%                    │
│ + 21종목 더 보기               │
│                                │
│ ─────────────────────────────  │
│ 연결된 섹터: 반도체 ● AI인프라 │  ← Chain Sight 미리보기
│ [관계 지도 열기]               │  ← Chain Sight 직접 진입
└────────────────────────────────┘
```

이렇게 하면 종목 상세 페이지를 거치지 않고 카드에서 바로 Chain Sight로 진입 가능하다.

**방안 B: 관계 카드를 독립 타입으로 분리 강화**

relation 카테고리 시그널(S1, S2, S3, S4)은 이미 Chain Sight DNA의 핵심이다.
이 카테고리 카드는 다른 카드와 다른 시각적 처리를 하라:

```
일반 카드: 흰색 테두리
관계 카드: 보라색(#A371F7) 테두리 + "Chain Sight" 워터마크
```

사용자가 보라색 카드 = Chain Sight 관련이라는 것을 패턴으로 학습하게 된다.
색상이 아닌 테두리 색상으로 구분하면 색맹 사용자도 문제없다.

**방안 C: 섹터 소외주 카드의 Chain Sight 자동 연결**

섹터 소외주(S2) 시그널은 Chain Sight 전환율이 42%로 가장 높다.
이 카드를 클릭하면 종목 리스트 대신 바로 Chain Sight 그래프를 보여줘라:

```
[섹터 소외주 카드 클릭]
→ "섹터 전체는 +5% 상승, 아직 못 따라간 종목들"
→ 그래프에서 섹터 노드와 소외 종목들의 관계 즉시 표시
→ 종목 리스트는 그래프 옆 사이드바로
```

이것이 진정한 Chain Sight DNA 적용이다.

### 5-3. 연쇄 발견 시나리오 설계

계획서에 명시된 플로우를 UX 관점에서 구체화:

```
시나리오: NVDA 거래량 폭발에서 AMD 소외주 발견까지

Step 1. 메인 대시보드
  "거래량 폭발 — 23종목" 카드에 NVDA +5.2% 보임
  카드 하단: "연결 섹터: 반도체 ● AI인프라"
  → 관계 지도 열기 탭

Step 2. Chain Sight (NVDA 중심)
  NVDA 노드 강조 + 반도체 섹터 노드 + AMD, TSM, INTC 연결
  AMD 노드가 흐릿하게 표시 (오늘 시그널 없음 = 소외)
  → AMD 노드 탭

Step 3. Ambient Hint 자동 노출
  AMD 노드 탭 시: "NVDA +5.2% 인데 AMD 오늘 -0.2%"
  → "섹터 소외주 시그널 감지됨" 배지 자동 표시
  → "AMD 모니터링 추가" 원탭 액션

Step 4. 발견 완료
  NVDA 거래량 폭발 → AMD 소외주 발견 → 모니터링 등록
  총 3탭, 페이지 전환 없음 (슬라이드 패널 레이어링)
```

이 시나리오를 실현하려면 Chain Sight 그래프에서 시그널 정보를 노드에 오버레이하는
백엔드/프론트엔드 연동이 필요하다. 이것은 Phase 2 구현 항목으로 계획서에 표시가 맞다.

4. **경쟁 앱 표준**: Bloomberg, TradingView, Robinhood 모두 다크모드 우선

## 7. 추가/수정 제안

### 7-1. 필수 추가: 날짜 및 데이터 신선도 표시

현재 계획에 날짜 표시가 시장 요약 바에 포함되어 있지 않다.
JSON Baking 아키텍처에서는 데이터가 새벽 4시에 구워지므로,
사용자가 "이 데이터가 언제 것인가"를 명확히 알아야 한다.

**권장:**

```
┌─────────────────────────────────────────────────┐
│ 2026.02.26 장 마감 기준 · 오전 4:08 업데이트     │  ← 상단 고정
└─────────────────────────────────────────────────┘
```

만약 파이프라인이 실패해 전일 데이터가 표시된다면:

```
┌─────────────────────────────────────────────────┐
│ 2026.02.25 데이터 (오늘 업데이트 지연 중)        │  ← 경고 스타일
└─────────────────────────────────────────────────┘
```

`meta.json`에서 `generated_at`을 읽어 표시하면 된다.
이것은 사용자 신뢰의 핵심이다. 투자 데이터에서 신선도 불명확은 치명적이다.

### 7-2. 필수 추가: 스켈레톤 로딩

JSON Baking으로 로드가 빠를 것이지만, 네트워크 상황에 따라 지연될 수 있다.
모든 카드에 정확한 레이아웃의 스켈레톤을 구현해야 한다.

```tsx
// shadcn/ui Skeleton 사용
<div className="grid grid-cols-4 gap-3">
  {Array.from({ length: 8 }).map((_, i) => (
    <div key={i} className="rounded-xl border border-border p-4">
      <Skeleton className="h-4 w-24 mb-2" />   {/* 타이틀 */}
      <Skeleton className="h-6 w-12 mb-4" />   {/* 종목 수 */}
      <Skeleton className="h-4 w-full mb-1" /> {/* 종목 1 */}
      <Skeleton className="h-4 w-full" />      {/* 종목 2 */}
    </div>
  ))}
</div>
```

스켈레톤은 실제 카드와 정확히 같은 크기여야 한다.
콘텐츠 로드 시 레이아웃이 뛰지 않아야 한다 (CLS = 0).

### 7-3. 권장 추가: 시그널 강도 시각화

현재 계획에는 시그널 수(23종목)만 있고 강도 표시가 없다.
모든 "거래량 3배 폭발"이 동등하지 않다.

**권장: 강도 인디케이터**

```
┌────────────────────────────┐
│ 거래량 폭발        23종목  │
│ ●●●●○  강도: 높음          │  ← 5점 척도 인디케이터
└────────────────────────────┘
```

강도는 해당 카테고리의 평균 시그널 값(vol_ratio, return_pct 등)으로 산출.
7일 평균 대비 오늘 강도를 점으로 표시.

### 7-4. 권장 추가: 시그널 카드 빈 상태 처리

계획서에 언급 없지만 중요한 엣지 케이스:
파이프라인은 정상이지만 특정 카테고리 시그널이 0개인 날.

**권장:**

```
┌────────────────────────────┐
│ 매집 의심              0종목 │
│                            │
│ 오늘은 감지된 종목 없음     │
│ 지난 7일 기준: 평균 5종목   │  ← 맥락 제공
└────────────────────────────┘
```

빈 카드를 숨기면 그리드가 흐트러진다. 빈 상태로 표시하되 맥락을 제공하라.

### 7-5. 권장 추가: 뉴스 match_type별 시각적 분기 강화

계획서에 match_type별 프론트엔드 처리가 명시되어 있다:

```
symbol_today  → 강조, 흰색 배경
symbol_7d     → 보통, 살짝 흐리게
industry_7d   → 이탤릭
profile       → 별도 스타일
```

이것을 구체적인 shadcn/ui 구현 스펙으로 정의:

```tsx
// match_type별 Badge 컴포넌트 스타일

symbol_today:  <Badge variant="default">오늘</Badge>         // 밝은 파랑
symbol_7d:     <Badge variant="secondary">3일 전</Badge>     // 회색
symbol_30d:    <Badge variant="outline">2주 전</Badge>       // 테두리만
industry_7d:   <Badge variant="outline" italic>섹터 뉴스</Badge>  // 이탤릭
profile:       텍스트만, 아이콘 없이 (📝 아이콘 제거)
```

특히 `profile` 타입은 뉴스가 아닌 종목 설명이므로,
뉴스 아이콘(📰)을 붙이면 안 된다. 명확한 구분이 신뢰도를 높인다.

### 7-6. 장기 권장: 개인화 레이어 설계 고려

현재 계획에서 개인화는 Phase 2+ 항목이다.
그러나 개인화 레이어를 나중에 끼워 넣으면 컴포넌트 재설계가 필요하다.
처음부터 "개인화 가능한 구조"로 설계할 것을 권장한다.

**초기 설계에 포함해야 할 구조:**

```typescript
// 시그널 카드 순서를 외부 설정으로 제어 가능하게
interface DashboardConfig {
  cardOrder: string[];          // ['volume', 'momentum', ...]
  pinnedCategories: string[];   // 상단 고정
  hiddenCategories: string[];   // 숨김
}

// 컴포넌트는 config를 받아 렌더링
<SignalCardGrid config={userConfig ?? defaultConfig} data={dashboardData} />
```

처음에는 `defaultConfig`로 모두 동일하게, 나중에 `userConfig`를 채우는 방식.
이 구조면 개인화 추가 시 컴포넌트를 건드리지 않아도 된다.

## 9. 구현 우선순위 요약

### Phase 1 MVP에 반드시 포함
