## 1. 설계 원칙

### 핵심 철학

```
"뉴스는 결과를 말해주고, EOD 스크리닝은 전조를 보여준다."
"남들 다 아는 뉴스 보고 들어가면 늦는데, 데이터는 이미 신호를 보내고 있었어."
```

### 5대 원칙

| #   | 원칙                               | 설명                                                                                                       |
| --- | ---------------------------------- | ---------------------------------------------------------------------------------------------------------- |
| 1   | **시그널이 주(主), 뉴스가 종(從)** | 시그널이 이상 현상을 먼저 포착하고, 뉴스는 "왜?"의 맥락을 사후에 부여한다. 뉴스로 종목을 찾게 하지 않는다. |
| 2   | **숫자가 아니라 스토리**           | RSI 28이 아니라 "5일 연속 빠졌다가 오늘 반등한 종목". 기술적 지표를 자연어 태그로 치환한다.                |
| 3   | **API 콜 0원**                     | 모든 시그널은 이미 DB에 있는 OHLCV로 로컬 계산 (pandas-ta). FMP 추가 과금 없음.                            |
| 4   | **JSON Baking**                    | 새벽 배치에서 모든 연산을 끝내고 정적 JSON으로 구워둔다. 사용자 접속 시 서버 부하 제로.                    |
| 5   | **관계(Chain Sight)로의 징검다리** | 모든 시그널은 개별 종목에서 끝나지 않고, 섹터/테마/공급망 관계 탐색으로 이어져야 한다.                     |

## 2. 시그널 체계

### 2-1. Tier 1 — 단순 계산, 즉시 구현

OHLCV 당일 + 최근 수일. 맥북에서 수 초 이내 전 종목 처리.

#### 가격 움직임

| ID  | 시그널명              | 계산 로직                                       | 자연어 태그                       |
| --- | --------------------- | ----------------------------------------------- | --------------------------------- |
| P1  | 연속 상승/하락        | N일 연속 close > prev_close                     | "5일 연속 상승 중 🔥"             |
| P2  | 수익률 상위           | 일간/주간/월간 수익률 랭킹                      | "오늘의 로켓 🚀"                  |
| P3  | 갭 감지               | open > prev_close × 1.03                        | "장 시작부터 3% 점프! 무슨 일이?" |
| P4  | 장대양봉/음봉         | abs(close-open)/open ≥ 3%, 꼬리 짧음            | "오늘 확실하게 방향을 정한 종목"  |
| P5  | 52주 신고가/저가 근접 | close ≥ 52w_high × 0.95                         | "52주 최고가 코앞! 돌파할까? 🏆"  |
| P6  | 라운드 피겨 돌파      | 종가가 $10/$50/$100 등 마디 돌파 또는 직전 정체 | "드디어 $100 고지를 넘은 종목"    |
| P7  | 저가 대비 반등률      | (close-low)/(high-low) ≥ 0.8                    | "장중 폭락 후 살아 돌아온 종목"   |

#### 거래량

| ID  | 시그널명         | 계산 로직                               | 자연어 태그                     |
| --- | ---------------- | --------------------------------------- | ------------------------------- |
| V1  | 거래량 이상 감지 | volume / avg_vol_20d ≥ 2.0              | "평소의 3배 거래량 폭발 💥"     |
| V2  | 신규 거래 활성   | avg_vol_20d < 10K → 오늘 > 100K         | "잠자던 종목이 갑자기 깨어났다" |
| V3  | 비정상적 변동성  | (high-low)/close가 20일 평균의 2배 이상 | "평소와 다른 움직임 감지"       |

#### 가격-거래량 복합 (돈의 흐름)

| ID  | 시그널명           | 계산 로직                                           | 자연어 태그                                  |
| --- | ------------------ | --------------------------------------------------- | -------------------------------------------- |
| PV1 | 가격-거래량 효율성 | ΔPrice% / (vol/avg_vol) 비율 급증                   | "가볍게 날아오르는 종목 (매물 공백)"         |
| PV2 | 매집 의심          | vol > avg×3 이면서 abs(close-open)/(high-low) < 0.2 | "누군가 아래에서 조용히 다 받아먹고 있다 🕵️" |
| PV3 | 장 마감 매집 패턴  | (close-low)/(high-low) > 0.7이 3일 연속             | "장 마감 직전마다 누군가 사들이는 종목"      |

### 2-2. Tier 2 — 수일~수주 히스토리 기반

최소 20~60일 히스토리. pandas rolling 벡터 연산.

#### 이동평균 & 추세

| ID  | 시그널명        | 계산 로직                         | 자연어 태그                                 |
| --- | --------------- | --------------------------------- | ------------------------------------------- |
| MA1 | 골든/데드크로스 | EMA5 > EMA20 상향돌파             | "골든크로스 발생 📈"                        |
| MA2 | 이동평균 이격도 | (close-SMA50)/SMA50 과매수/과매도 | "50일 평균에서 너무 멀리 왔다 (과열 주의)"  |
| MA3 | 이동평균 수렴   | 5/10/20/50일 SMA가 ±2% 범위 밀집  | "곧 방향이 정해진다 — 투자자들의 합의 구간" |
| MA4 | 추세 전환 감지  | 하락 추세에서 EMA5 > EMA20 전환   | "하락→상승 추세 전환 시작"                  |

#### 패턴

| ID  | 시그널명              | 계산 로직                                            | 자연어 태그                                 |
| --- | --------------------- | ---------------------------------------------------- | ------------------------------------------- |
| PT1 | 지지/저항선 돌파      | close > rolling_max(high, 60d)                       | "2개월 만에 최고가 갱신 🏆"                 |
| PT2 | 가격 압축             | 20일 변동폭 표준편차 역대 하위 10%                   | "조용하다가 곧 터질 수 있는 종목 💣"        |
| PT3 | N일 만에 첫 양봉      | 5일+ 연속 하락 후 첫 양봉                            | "5일 연속 하락 후 첫 반등! 반전의 시작?"    |
| PT4 | 바닥 다지기           | 60일 신저가 후, 5일간 저가 미갱신 + 거래량 미세 증가 | "바닥을 다진 복수혈전 종목 🥊"              |
| PT5 | 전고점 매물대 소화    | 3~6개월 고점 부근 횡보 후 거래량 동반 돌파           | "6개월 전 물량을 이겨내고 다시 달리는 종목" |
| PT6 | 거래량 동반 상승 연속 | 5일 중 4일+ (close>prev) AND (vol>prev_vol)          | "돈이 몰리면서 오르고 있다 💪"              |

#### 섹터/그룹 상대 강도 (→ Chain Sight 직접 연결)

| ID  | 시그널명        | 계산 로직                                | 자연어 태그                                      |
| --- | --------------- | ---------------------------------------- | ------------------------------------------------ |
| S1  | 섹터 상대 강도  | 종목 5일 수익률 vs 섹터 평균             | "섹터가 빠지는데 이 종목만 버티고 있다"          |
| S2  | 섹터 소외주     | 동일 industry 5일 평균 +10%인데 혼자 0%  | "친구들은 다 갔는데, 아직 출발 안 한 종목 🐢→🐇" |
| S3  | 대장주 따라잡기 | industry 시총 1위 +5%인데 하위 종목 미동 | "형님들이 가는데 아직 출발 안 한 종목"           |
| S4  | 폭락장 생존자   | SPY -2%+ 하락일에 보합 또는 상승         | "폭풍 속 흔들리지 않는 종목 🌲"                  |

### 2-3. Tier 3 — 정교한 지표 계산

pandas-ta / ta-lib 벡터 연산. 전 종목 수 분 이내.

#### 기술적 지표

| ID  | 시그널명          | 계산 로직                        | 자연어 태그                             |
| --- | ----------------- | -------------------------------- | --------------------------------------- |
| T1  | RSI 과매도/과매수 | RSI(14) < 30 또는 > 70           | "너무 빠져서 반등 가능성이 보이는 종목" |
| T2  | 볼린저 밴드 이탈  | close < BB_lower 또는 > BB_upper | "볼린저 하단 이탈 — 반등 구간?"         |
| T3  | MACD 크로스       | MACD선 > 시그널선 상향돌파       | "MACD 매수 시그널 📊"                   |
| T4  | ADX 추세 강도     | ADX > 25 강한 추세               | "확실한 추세가 잡힌 종목"               |

#### 고급 복합

| ID  | 시그널명               | 계산 로직                        | 자연어 태그                               |
| --- | ---------------------- | -------------------------------- | ----------------------------------------- |
| A1  | OBV 다이버전스         | 가격 횡보/하락 + OBV 상승        | "스마트머니가 조용히 쌓이고 있다 🧠"      |
| A2  | 멀티 타임프레임 스코어 | 1주+1개월+3개월 수익률 종합 점수 | "3개 시간대 모두 상승 — 탄탄한 모멘텀 🎯" |
| A3  | 변동성 조정 수익률     | 수익률/표준편차 랭킹 (간이 샤프) | "흔들림 없이 꾸준히 오르는 우등생 📐"     |
| A4  | 캔들 패턴              | 해머, 도지, 인걸핑 (ta-lib)      | "반전 캔들 '해머' 출현 🔨"                |

#### 뉴스 연동 (사후 해석)

| ID  | 시그널명         | 계산 로직                                                       | 자연어 태그                                     |
| --- | ---------------- | --------------------------------------------------------------- | ----------------------------------------------- |
| N1  | 시간차 공격      | 3일 전 뉴스 폭발 + 당시 무반응 → 오늘 거래량 급증               | "3일 전 소문이 이제야 돈으로 바뀌고 있다 ⏰→💰" |
| N2  | 시그널 맥락 부여 | 시그널 종목에 대해 계층적 뉴스 탐색 (당일→7일→30일→섹터→프로필) | "거래량 3배 폭증 — 관련: [FDA 승인 임박]"       |

### 2-4. 시그널 카테고리 체계

프론트엔드 카드 그룹핑 및 사용자 선호 추적용.

```
momentum    — 가격 움직임, 연속 상승/하락, 수익률 (P1, P2, P3, P4)
volume      — 거래량 이상, 매집, 신규 활성 (V1, V2, V3, PV1, PV2, PV3)
breakout    — 52주 신고가, 저항선 돌파, 라운드 피겨 (P5, P6, PT1, PT5)
reversal    — 반등, 추세 전환, 캔들 패턴, 바닥 다지기 (P7, PT3, PT4, A4)
relation    — 섹터 소외, 대장 따라잡기, 상대 강도 (S1, S2, S3, S4)
technical   — RSI, MACD, BB, 골든크로스, 이평선 (MA1~4, T1~4)
composite   — 멀티 타임프레임, OBV, 변동성 조정 (A1, A2, A3)
```

## 3. 배치 파이프라인 아키텍처

### 핵심 설계: Pre-computation + JSON Baking

사용자 접속 시 DB 쿼리를 하지 않는다.
새벽 배치에서 모든 연산을 끝내고 정적 JSON으로 구워둔다.
Next.js 프론트엔드는 이 JSON만 fetch하여 화면을 그린다.

### 전체 흐름도

```
04:00 KST (미장 마감 ~1시간 후)
│
▼
┌─────────────────────────────────────────────────────┐
│ Stage 1: Data Ingestion (데이터 수신)                  │
│ ──────────────────────────────────                    │
│ • FMP Bulk EOD API 호출 (1회, 100 API calls)          │
│ • 전체 US 거래소 OHLCV + sector/industry 수신          │
│ • PostgreSQL raw_eod_daily 테이블에 적재               │
│ • SPY/QQQ/VIX 등 지수 데이터도 함께 저장               │
│                                                     │
│ 소요: ~30초 │ 서버 부하: 낮음 (네트워크 I/O)            │
└──────────────────────┬──────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────┐
│ Stage 2: Hard Filter (잡주 제거)                      │
│ ──────────────────────────────────                    │
│ • 시가총액 하위 20% 제외                               │
│ • 일 거래대금 $100K 미만 제외                           │
│ • OTC/핑크시트 등 거래소 필터                           │
│ • ~10,000 → ~6,000종목으로 축소                        │
│                                                     │
│ 소요: <1초 │ 서버 부하: 최소                            │
└──────────────────────┬──────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────┐
│ Stage 3: Vectorized Calculation (벡터 연산)            │
│ ──────────────────────────────────                    │
│ • PostgreSQL에서 최근 200일치 OHLCV를 DataFrame 로드    │
│ • pandas-ta로 전 종목 일괄 계산:                       │
│   - RSI(14), SMA(50,200), EMA(5,20)                  │
│   - MACD(12,26,9), Bollinger Bands(20,2)             │
│   - OBV, ADX                                        │
│ • Tier 1 단순 계산 (연속 상승, 거래량 비율, 갭 등)      │
│ • Tier 2 패턴 감지 (지지/저항 돌파, 압축 등)            │
│ • 섹터/인더스트리 평균 수익률 계산 + 상대 강도           │
│                                                     │
│ ⚠️ 핵심: For-loop 절대 금지. 모두 벡터 연산.           │
│                                                     │
│ 소요: 1~3분 │ 서버 부하: CPU 집중 (단발성)              │
└──────────────────────┬──────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────┐
│ Stage 4: Signal Tagging (스토리 태깅)                  │
│ ──────────────────────────────────                    │
│ • 계산 결과 기반 룰 엔진으로 태그 부여                   │
│ • 각 태그에 카테고리, 우선순위, 자연어 메시지 매핑        │
│ • daily_signals 테이블에 JSONB로 저장                   │
│ • signal_accuracy 테이블에 추적 레코드 생성 (수익률 NULL)│
│                                                     │
│ 소요: <30초 │ 서버 부하: 낮음                           │
└──────────────────────┬──────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────┐
│ Stage 5: News Enrichment (뉴스 축적 + 계층적 매칭)     │
│ ──────────────────────────────────                    │
│                                                     │
│ 5-A. 뉴스 일일 수집 (백그라운드 축적)                   │
│   • FMP Stock News API → 최근 24시간 뉴스 bulk 수집    │
│   • stock_news 테이블에 적재 (symbol, headline,        │
│     sector, industry, published_at)                  │
│   • 시간이 지날수록 뉴스 DB 풍부해짐                    │
│   • API 콜: 수십 건/일                                │
│                                                     │
│ 5-B. 시그널 종목 뉴스 매칭 (계층적 시간창 탐색)          │
│   • 시그널 발생 종목별로 아래 순서로 탐색:              │
│     ① symbol 매칭 (오늘)          → match_type: today │
│     ② symbol 매칭 (최근 7일)      → match_type: 7d    │
│     ③ symbol 매칭 (최근 30일)     → match_type: 30d   │
│     ④ 같은 industry 뉴스 (7일)   → match_type: sector│
│     ⑤ 종목 프로필 기반 한 줄 생성  → match_type: profile│
│   • ※ "관련 뉴스 없음"은 절대 표시하지 않음            │
│   • ※ 중소형주도 최소 섹터 뉴스 또는 프로필은 보장      │
│                                                     │
│ 소요: 1~2분 │ API 콜: 최소 (축적분은 DB에서 조회)       │
└──────────────────────┬──────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────┐
│ Stage 6: JSON Baking (정적 파일 생성) ★ 핵심 ★         │
│ ──────────────────────────────────                    │
│ • 모든 연산 결과를 정적 JSON 파일로 구워냄               │
```

### 뉴스 매칭 로직: 계층적 시간창 탐색

**원칙: "관련 뉴스 없음"은 절대 표시하지 않는다.**

중소형주는 매일 뉴스가 나오지 않으므로, 시간창을 점진적으로 확장하여 반드시 맥락을 채운다.

```
탐색 순서 (처음 매칭되는 단계에서 중단):

① symbol 매칭 (당일)
   stock_news에서 symbol = 'PLTR' AND published_at = today
   → 있으면: 📰 "헤드라인" — 오늘

② symbol 매칭 (최근 7일)
   stock_news에서 symbol = 'PLTR' AND published_at >= today - 7d
   → 있으면: 📰 "헤드라인" — 3일 전

③ symbol 매칭 (최근 30일)
   stock_news에서 symbol = 'PLTR' AND published_at >= today - 30d
   → 있으면: 📰 "헤드라인" — 2주 전

④ 같은 industry 뉴스 (최근 7일)
   stock_news에서 industry = 'Software—Infrastructure' AND published_at >= today - 7d
   → 있으면: 📰 "헤드라인" — 섹터 뉴스

⑤ 종목 프로필 기반 한 줄 생성 (최후 수단)
   company_profile에서 description 요약
   → 📝 "AI 기반 데이터 분석 플랫폼. 시총 $180B, 매출 YoY +28%"
```

프론트엔드에서 `match_type`에 따라 표시 스타일 분기:

```
symbol_today  → 📰 "헤드라인" — 오늘          (강조, 흰색 배경)
symbol_7d     → 📰 "헤드라인" — 3일 전        (보통, 살짝 흐리게)
symbol_30d    → 📰 "헤드라인" — 2주 전        (보통, 흐리게)
industry_7d   → 📰 "헤드라인" — 섹터 뉴스     (아이콘 다름, 이탤릭)
profile       → 📝 "프로필 한 줄"              (별도 스타일, 차분한 톤)
```

시간이 지날수록 `stock_news` 테이블에 뉴스가 축적되므로, ④⑤에 도달하는 종목이 자연스럽게 줄어든다.

## 6. DB 스키마

### 핵심 테이블

```sql
-- ============================================================
-- 1. 원본 EOD 데이터
-- ============================================================
CREATE TABLE raw_eod_daily (
    id              BIGSERIAL PRIMARY KEY,
    trade_date      DATE NOT NULL,
    symbol          VARCHAR(10) NOT NULL,
    open            DECIMAL(12,4),
    high            DECIMAL(12,4),
    low             DECIMAL(12,4),
    close           DECIMAL(12,4),
    adjusted_close  DECIMAL(12,4),
    volume          BIGINT,

    -- FMP bulk에서 제공하는 메타 정보
    sector          VARCHAR(50),
    industry        VARCHAR(100),
    market_cap      BIGINT,

    created_at      TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(trade_date, symbol)
);

CREATE INDEX idx_eod_date ON raw_eod_daily(trade_date);
CREATE INDEX idx_eod_symbol ON raw_eod_daily(symbol, trade_date);
CREATE INDEX idx_eod_sector ON raw_eod_daily(trade_date, sector);

-- ============================================================
-- 2. 일별 시그널 (배치 결과)
-- ============================================================
CREATE TABLE daily_signals (
    id              BIGSERIAL PRIMARY KEY,
    signal_date     DATE NOT NULL,
    symbol          VARCHAR(10) NOT NULL,

    -- 비정규화된 EOD (빠른 조회용)
    close           DECIMAL(12,4),
    change_pct      DECIMAL(8,4),
    volume          BIGINT,

    -- 주요 계산 지표
    rsi_14          DECIMAL(6,2),
    sma_50          DECIMAL(12,4),
    sma_200         DECIMAL(12,4),
    ema_5           DECIMAL(12,4),
    ema_20          DECIMAL(12,4),
    macd            DECIMAL(10,4),
    macd_signal     DECIMAL(10,4),
    bb_upper        DECIMAL(12,4),
    bb_lower        DECIMAL(12,4),
    obv             BIGINT,
    vol_ratio_20d   DECIMAL(6,2),
    close_position  DECIMAL(4,2),     -- (close-low)/(high-low)
    momentum_score  DECIMAL(4,2),

    -- 시그널 태그 (JSONB)
    tags            JSONB DEFAULT '[]',
    -- 예: ["volume_spike_3x", "golden_cross", "sector_laggard"]

    -- 태그 상세 (우선순위, 메시지 포함)
    tag_details     JSONB DEFAULT '{}',

    -- 메타
    sector          VARCHAR(50),
    industry        VARCHAR(100),
    market_cap      BIGINT,

    created_at      TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(signal_date, symbol)
);

CREATE INDEX idx_signals_date ON daily_signals(signal_date);
CREATE INDEX idx_signals_tags ON daily_signals USING GIN(tags);
CREATE INDEX idx_signals_sector ON daily_signals(signal_date, sector);

-- ============================================================
-- 3. 시그널 정확도 추적 (ML 학습 데이터)
-- ============================================================
CREATE TABLE signal_accuracy (
    id              BIGSERIAL PRIMARY KEY,
    signal_date     DATE NOT NULL,
    symbol          VARCHAR(10) NOT NULL,
    signal_tag      VARCHAR(50) NOT NULL,
    signal_value    DECIMAL(10,4),

    -- 시그널 발생 시점 맥락
    close_at_signal DECIMAL(12,4),
    market_cap      BIGINT,
    sector          VARCHAR(50),
    vix_at_signal   DECIMAL(6,2),
    spy_change_pct  DECIMAL(6,2),

    -- 이후 수익률 (배치로 매일 업데이트)
    return_1d       DECIMAL(8,4),
    return_5d       DECIMAL(8,4),
    return_20d      DECIMAL(8,4),

    -- 시장 대비 초과 수익률
    excess_1d       DECIMAL(8,4),
    excess_5d       DECIMAL(8,4),
    excess_20d      DECIMAL(8,4),

    created_at      TIMESTAMPTZ DEFAULT NOW(),
    updated_at      TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_acc_tag ON signal_accuracy(signal_tag);
CREATE INDEX idx_acc_date ON signal_accuracy(signal_date);
CREATE INDEX idx_acc_backfill ON signal_accuracy(signal_date, return_1d)
    WHERE return_1d IS NULL;

-- ============================================================
-- 4. 뉴스 축적 DB (계층적 뉴스 매칭용)
-- ============================================================
CREATE TABLE stock_news (
    id              BIGSERIAL PRIMARY KEY,
    symbol          VARCHAR(10),          -- NULL 가능 (섹터/산업 일반 뉴스)
    published_at    TIMESTAMPTZ NOT NULL,
    headline        TEXT NOT NULL,
    snippet         TEXT,
    source          VARCHAR(100),
    url             TEXT,

    -- 매칭용 메타
    sector          VARCHAR(50),
    industry        VARCHAR(100),
    keywords        JSONB DEFAULT '[]',   -- 추출된 키워드

    created_at      TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_news_symbol ON stock_news(symbol, published_at DESC);
CREATE INDEX idx_news_industry ON stock_news(industry, published_at DESC);
CREATE INDEX idx_news_date ON stock_news(published_at DESC);

-- ※ 뉴스 매칭 우선순위:
--   1순위: symbol 매칭 (당일)         → match_type: symbol_today
--   2순위: symbol 매칭 (최근 7일)     → match_type: symbol_7d
--   3순위: symbol 매칭 (최근 30일)    → match_type: symbol_30d
--   4순위: 같은 industry 뉴스 (7일)   → match_type: industry_7d
--   5순위: 종목 프로필 기반 한 줄 생성 → match_type: profile
-- ※ "관련 뉴스 없음"은 절대 표시하지 않음

-- ============================================================
-- 5. 파이프라인 실행 로그
-- ============================================================
CREATE TABLE pipeline_log (
    id              BIGSERIAL PRIMARY KEY,
    run_date        DATE NOT NULL,
    started_at      TIMESTAMPTZ NOT NULL,
    completed_at    TIMESTAMPTZ,

    -- 각 Stage 소요시간 (초)
    stage_ingest_sec    DECIMAL(6,1),
    stage_filter_sec    DECIMAL(6,1),
    stage_calculate_sec DECIMAL(6,1),
    stage_tag_sec       DECIMAL(6,1),
    stage_news_sec      DECIMAL(6,1),
    stage_bake_sec      DECIMAL(6,1),
    stage_backfill_sec  DECIMAL(6,1),
    total_sec           DECIMAL(8,1),

    -- 결과 통계
    stocks_ingested     INTEGER,
    stocks_filtered     INTEGER,
    signals_generated   INTEGER,
    stocks_with_signals INTEGER,
    news_matched        INTEGER,
    json_files_created  INTEGER,

    -- 뉴스 매칭 분포 (계층별 건수)
    news_match_distribution JSONB,
    -- {"symbol_today": 32, "symbol_7d": 412, "symbol_30d": 287,
    --  "industry_7d": 98, "profile": 18}

    -- 상태
    status          VARCHAR(20) DEFAULT 'running',  -- running, success, failed
    error_message   TEXT,

    -- 시그널 분포 스냅샷
    signal_distribution JSONB,
    -- {"momentum": 45, "volume": 23, ...}

    created_at      TIMESTAMPTZ DEFAULT NOW()
);

-- ============================================================
-- 6. 사용자 행동 로그 (개인화 + ML용)
-- ============================================================
CREATE TABLE user_signal_interactions (
    id              BIGSERIAL PRIMARY KEY,
    user_id         INTEGER REFERENCES auth_user(id),
    interaction_at  TIMESTAMPTZ DEFAULT NOW(),

    action_type     VARCHAR(20),
    -- 'card_click': 시그널 카드 클릭
    -- 'stock_click': 종목 리스트에서 종목 클릭
    -- 'chain_sight': Chain Sight 진입
    -- 'monitor_add': Node Monitoring 등록
    -- 'portfolio_add': Portfolio 추가

    signal_category VARCHAR(30),   -- 'momentum', 'volume' 등
    signal_tag      VARCHAR(50),   -- 'volume_spike_3x' 등
    symbol          VARCHAR(10),

    session_id      UUID,
    source_page     VARCHAR(50)    -- 'dashboard', 'card_detail', 'stock_detail'
);

CREATE INDEX idx_interact_user ON user_signal_interactions(user_id, interaction_at);
CREATE INDEX idx_interact_category ON user_signal_interactions(signal_category, interaction_at);
```

## 7. ML 확장 로드맵

### ML이 가치를 만드는 4개 영역

```
① 시그널 품질 향상     "어떤 시그널이 실제로 맞았는가"
② 시그널 조합 최적화   "시그널이 겹치면 더 강한가"
③ 개인화 추천          "이 사용자에게 맞는 시그널은"
④ 새로운 패턴 발견     "사람이 못 보는 패턴 찾기"
```

### 타임라인

#### Month 1-3: 데이터 축적기 (ML 없음)

```
할 일:
  • 룰 기반 시그널 운영 및 안정화
  • daily_signals 매일 적재
  • signal_accuracy 매일 backfill (1d/5d/20d 수익률)
  • user_signal_interactions 클릭 로그 수집

축적 목표:
  • signal_accuracy: ~60일 × ~150건/일 = ~9,000 행
  • 충분하지 않지만, 시그널별 기초 통계는 가능

이 기간의 관리자 액션:
  • signal_accuracy CLI로 주 1회 점검
  • 명백히 부정확한 시그널 임계값 수동 조정
  • 교육 메시지/카드 네이밍 사용자 반응 보고 개선
```

#### Month 3-6: 통계 기반 개선

```
적용:
  ① z-score 기반 동적 임계값
    • 종목별/섹터별 거래량 z-score → 고정 2.0x 대신 동적
    • VIX 구간별 임계값 분기 (고변동성 시장 → 더 강한 시그널만)

  ② 정확도 기반 가중치 수동 조정
    • signal_accuracy 데이터 기반
    • 수익률 우수 시그널(RSI 과매도, 매집) → 카드 우선순위 상향
    • 수익률 약한 시그널(연속5일상승, 갭업) → 경고 문구 추가

  ③ 사용자 클릭 기반 카드 정렬
    • 최근 30일 클릭 빈도로 카드 순서 개인화
    • 규칙 기반 (ML 아님)

축적 목표:
  • signal_accuracy: ~120일 × ~150건/일 = ~18,000 행
```

#### Month 6-12: 기본 ML 도입

```
적용:
  ① Gradient Boosting (시그널 품질 스코어링)
    Features: signal_value, market_cap, sector, vix, spy_change,
              vol_ratio, rsi, 시장 레짐 등
    Target:   return_20d > market_return_20d (초과수익 여부)
    모델:     XGBoost / LightGBM
    용도:     각 시그널에 "신뢰도 점수" 부여
              → 프론트에서 ⭐⭐⭐ 같은 강도 표시

  ② 시그널 조합 효과 학습
    Features: 각 시그널 유무를 binary feature로
              has_volume_spike, has_golden_cross, ... (다중 태그 종목)
    Target:   초과수익 여부
    모델:     Random Forest (feature importance로 조합 해석)
    용도:     "거래량 폭발 + RSI 과매도 조합은 단독보다 2배 강력"

  ③ Content-based 추천
    사용자의 과거 클릭 패턴과 유사한 시그널 우선 노출
    간단한 유사도 기반 (사용자 수 적으므로 collaborative 불가)

축적 목표:
  • signal_accuracy: ~240일 × ~150건/일 = ~36,000 행
  • 모델 학습에 충분한 데이터
```

#### Month 12+: 고급 ML

```
적용:
  ④ 패턴 클러스터링 (Unsupervised)
    • 전 종목의 N일간 OHLCV 시퀀스를 벡터화
    • K-means / DBSCAN으로 유사 패턴 그룹핑
    • "이 클러스터에 속한 종목은 이후 평균 X% 상승" 발견
    • 사람이 이름 붙이지 않은 새로운 패턴

  ⑤ 시계열 이상 탐지 (Anomaly Detection)
    • Isolation Forest / Autoencoder
    • "이 종목의 오늘 움직임은 과거 패턴과 확연히 다르다"
    • 기존 룰에 안 걸리는 이상 징후 포착

  ⑥ 시그널 시퀀스 예측
    • LSTM / Transformer
    • "거래량 폭발 → 3일 횡보 → ?" 다음 스텝 예측
    • Node Monitoring에 "내일 이런 시그널이 나올 확률 X%"

  ⑦ 사용자 세그먼트 기반 맞춤 추천
    • 투자 성향 자동 분류 (모멘텀파, 역발상파, 가치투자파)
    • 세그먼트별 시그널 카드 세트 최적화
```

### ML 개선 여지별 시그널 분류

```
높은 ML 개선 여지 (판단이 필요한 것):
  • PV2 매집 의심      → "진짜 매집 vs 단순 횡보" 구분
  • S2  섹터 소외주     → "곧 따라갈 종목 vs 뒤처진 이유 있는 종목"
  • PT4 바닥 다지기     → "진짜 바닥 vs 더 빠질 바닥"
  • A2  모멘텀 스코어   → 최적 가중치 + 시장 상황별 조합

낮은 ML 개선 여지 (룰로 충분한 것):
  • P1  연속 상승/하락  → 단순 팩트
  • P5  52주 신고가     → 객관적 수치
  • MA1 골든/데드크로스 → 수학적 정의 명확
  • P6  라운드 피겨     → 계산 명확
```

## 8. 구현 페이즈

### Phase 1 — MVP (2~3주)

스토리가 강하고 계산이 단순한 시그널 + 핵심 파이프라인.

```
백엔드:
  • FMP Bulk EOD 수신 → raw_eod_daily 적재
  • Hard Filter + Tier 1 시그널 계산 (P1, P3, P5, V1, S4)
  • Signal Tagging → daily_signals 적재
  • JSON Baking → dashboard.json + cards/*.json 생성
  • 크론탭 등록 (매일 04:00 KST)
  • pipeline_status CLI

프론트:
  • 메인 대시보드 (시장 요약 + 5개 시그널 카드)
  • 카드 클릭 → 종목 리스트
  • 종목 클릭 → 기존 Chain Sight 연결

DB:
  • raw_eod_daily, daily_signals, pipeline_log 생성
  • signal_accuracy 테이블 생성 + backfill 배치 포함
```

### Phase 2 — 관계 연결 (3~6주)

Chain Sight로 이어지는 시그널 + 뉴스 맥락 + 모니터링 연결.

```
백엔드:
  • Tier 2 시그널 추가 (S1, S2, S3, MA1, PT1, PT4, PT6, PV2)
  • News Enrichment Stage (뉴스 일일 수집 + 계층적 매칭)
  • stocks/{SYMBOL}.json 개별 종목 시그널 히스토리 생성
  • signal_quality, signal_accuracy CLI

프론트:
  • 시그널 카드 8개로 확장
  • 종목 상세 → 시그널 타임라인 뷰
  • [Chain Sight 🔗] [모니터링+] 버튼 연결
  • 뉴스 맥락 표시 (시그널이 주, 뉴스가 종)
  • 교육 팝업 (시그널 설명 접기/펼치기)
  • 메인 대시보드에 Chain Sight 미니맵 (선택)

DB:
  • user_signal_interactions 테이블 생성 + 클릭 로그 수집 시작
```

### Phase 3 — 정교화 + ML 준비 (1~3개월)

Tier 3 지표 + 통계 기반 개선 + 개인화 시작.

```
백엔드:
  • Tier 3 시그널 추가 (T1~T4, A1~A4)
  • z-score 동적 임계값
  • VIX 기반 시장 레짐별 임계값 분기
  • 정확도 데이터 기반 시그널 가중치 조정
  • Django Admin 커스텀 뷰 (관리자 대시보드 웹화)

프론트:
  • 사용자 클릭 빈도 기반 카드 순서 개인화
  • 시그널 강도 표시 (정확도 높은 시그널 강조)
  • "추격 매수 주의" 등 맥락 경고 메시지
  • Node Monitoring에서 시그널 변화 알림

ML:
  • signal_accuracy 데이터 분석 시작
  • XGBoost 시그널 스코어링 모델 프로토타입
```

### Phase 4 — ML 본격 도입 (6개월+)

```
  • Gradient Boosting 시그널 품질 스코어
  • 시그널 조합 효과 학습
  • Content-based 추천
  • 패턴 클러스터링 (데이터 충분 시)
  • 메인 대시보드에 풀 히트맵 옵션 제공
  • 체인/테마 단위 모니터링 (요약형)
```
