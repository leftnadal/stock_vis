#### 캐싱 전략 적용

**실시간 가격 (`/quote`)**:

```

          300초 (장 마감 후)

캐시 우선순위:
1. Redis 캐시 (1차)
2. DB 캐시 (2차, 15분 이상 지난 데이터)
3. API 호출 (3차)
```

**계산**:

```
포트폴리오 조회 (사용자 접속 시마다):
- 캐시 히트율: 85% (60초 TTL)
- 필요한 API 호출: 1,440 × 15% = 216회/일

포트폴리오 초기 로드 (회사 정보):
- 캐시 히트율: 95% (1시간 TTL)
- 필요한 API 호출: 24 × 5% = 1.2회/일 ≈ 1회/일

**합계: ~217회/일** (단일 사용자 × 10종목)
```

**배치 API 활용** (선택적):

```
FMP 배치 API: `/batch-request-end-of-day`
- 최대 50개 종목을 1개 API로 요청
- 호출 감소: 10회 → 1회 (90% 절감)

만약 50개 종목으로 확대 시:
- 기존 예상: 217 × 5 = 1,085회
- 배치 적용: 250 (일일 한도 내)
```

```
- 종목 상세: 2회 × 10% (미스) = 0.2회
- 재무제표: 3회 (항상 최신) = 3회
```

```python
def get_cache_ttl(current_time, market_status):
```

## 7. 에러 처리 및 재시도 전략

### 7.1 API 호출 실패 시나리오

| 상황 | 원인 | 대응 | 호출 추가 | 캐시 사용 |
|-----|-----|------|---------|----------|
| API 일시 에러 (429) | Rate limiting | 1초 대기 후 재시도 | 1회 | 기존 데이터 |
| 서버 에러 (5xx) | FMP 서버 문제 | 지수 백오프 (최대 3회) | 3회 | 기존 데이터 |
| 네트워크 타임아웃 | 연결 문제 | 1초 대기 후 재시도 | 1회 | DB 폴백 |
| 데이터 누락 | API 미지원 | 에러 로깅 | 0회 | 기존 데이터 |

### 7.2 Fallback 전략 (호출 비용 절감)

```python
# 3단계 Fallback 체계
def get_stock_price(symbol):
    # 1차: Redis 캐시
    cached = cache.get(f"quote_{symbol}")
    if cached:
        return cached

    # 2차: 데이터베이스 (15분 이내 데이터)
    recent = Stock.objects.filter(
        symbol=symbol,
        last_updated__gte=now() - timedelta(minutes=15)
    ).first()
    if recent:
        return recent

    # 3차: API 호출 (캐시 + DB 미스)
    api_data = fmp_client.get_quote(symbol)
    cache.set(f"quote_{symbol}", api_data, timeout=60)
    return api_data
```

**호출 절감 효과**: 캐시 85% + DB 10% = API 호출 5%만 필요
