# 지시서 — NT-8 · Daily Report 뉴스 지표 퍼널 재구성

## 목적 / 현재 위치

- 문제: 보고서가 "LLM 분석률 = 분석 ÷ 전체 신규"로 단일 비율을 노출 → 분모(전체 신규)와 설계(Tier A+ 한정 deep)가 어긋나 "1% = critical" 착시. 실제 실행 경로는 정상(어제 통과 3 = 분석 3).
- 조치: 단일 비율을 **퍼널(N→M→K→J)** 로 재구성, **실행 건강(J/K)** 과 **커버리지(K/N)** 를 분리 표시. null 기록률(M/N)도 노출해 NT-2b 레버를 가리킴.
- 분류: 표시/집계 보정. **점수·분류·임계 로직은 건드리지 않음(그건 NT-2b).** 본 작업은 읽기 전용 집계 + 보고서 렌더만.

## ⛔ 절대 규칙 (위반 시 HALT·보고)

1. **prod DB write 금지.** 읽기 전용 집계만.
2. **점수화/분류/임계 로직 무변경.** Tier A 임계(0.70)는 _읽기만_ — 변경은 NT-2b.
3. **보고서 생성 코드가 git 밖 nightly 스크립트 영역이면 AI 무수정** → 사용자 수동 블록으로 *제시*만(실행은 사용자).
4. **news 도메인 모델 직접 접근이 shared 경계를 넘으면 HALT.** 집계는 합법 경로(shared 래퍼 또는 해당 앱 영역)로만.
5. main 직접 조작 금지. 작업은 worktree(`monorepo/sess-mgmt` 계열)에서.

## STEP 0 — 박지 말고 실측 (분기 결정용)

> 메모리 추정 금지. 아래로 "지표가 어디서 만들어지는지"를 확정하고 A/B/C 분기를 고른다.

1. 문자열·계산 위치:
   `grep -rn "분석률\|analysis_rate\|LLM 분석\|신규.*분석" ~/stock-vis-nightly/ <repo경로>`
   → git **밖** nightly 스크립트인가, git **안** 코드인가.
2. 분자/분모 소스: "신규 N"·"분석 J"를 세는 실제 쿼리/함수. news 모델 직접 접근인지, shared 경유인지.
3. Tier A 임계(0.70)가 상수로 사는 위치 — K 단계 라벨에 _읽어서_ 쓸 것(하드코딩·변경 금지).
4. importance_score **null 집계** 가능 여부(M 단계용).
5. 윈도 정의: 보고서가 "오늘(run까지)" / 24h / 48h 중 무엇을 N으로 쓰는가. 새 퍼널도 동일 윈도로.
6. 보고서 조립 방식(템플릿/f-string/섹션 마커)과 삽입 지점.

### 분기 (STEP 0 결과로 택1)

- **분기 A — git 밖 nightly 스크립트가 지표를 조립** → 본 지시서를 **사용자 수동 블록**으로 변환해 제시(AI 무수정). 집계 헬퍼만 git 안 읽기전용으로 둘지 여부도 STEP 0 위치로 판단.
- **분기 B — git 안 ops 보고 로직** → ops Claude Code로 진행(이 지시서 그대로).
- **분기 C — news 도메인 집계 함수 신설 필요** → 집계부는 **app:news 착수 스텁**으로 라우팅(읽기 전용 분석 헬퍼), 표시부만 ops. 섞지 말 것.
  → 분기가 모호하면 HALT·보고.

## 타겟 설계 (목업)

```
## 뉴스 분석 퍼널 (<날짜>, <윈도>)
신규 수집            N
 └ 점수 기록         M   (null N-M · X% 미기록 ⚠️ NT-2b)
    └ Tier A+ 통과   K   (임계 ≥<임계상수>)
       └ deep 분석   J

실행 건강 (분석/통과)   J/K   ※K=0이면 "N/A (통과 0) — critical 아님"
커버리지 (통과/신규)    K/N   ※Tier A+ 한정 설계, 낮음 정상
점수 기록률            M/N   ※임계 초과 null이면 ⚠️ + NT-2b 포인터
▸ 어제 대비: 통과 Kprev→K, deep Jprev→J, 기록률 …  (🆕/⬆️/➡️)
```

## 집계 정의 (정확히)

- **N 신규수집** = 윈도 내 신규 뉴스 row 수.
- **M 점수기록** = N 중 `importance_score IS NOT NULL`.
- **K Tier A+ 통과** = M 중 `importance_score >= <STEP0에서 읽은 임계>`. (하드코딩 금지)
- **J deep 분석완료** = `analyze_news_deep` 성공 결과 수(결과 테이블 ground truth 기준 — 어느 테이블인지 STEP 0).
- **실행 건강** = J/K. **K=0이면 0으로 나누지 말고 "N/A (통과 0)"** 표기.
- **커버리지** = K/N. **점수 기록률** = M/N.
- 윈도는 STEP 0에서 확인한 보고서 기존 윈도와 **반드시 일치**.

## 표시 규칙

- K=0 → 실행 건강 "N/A", **critical 아님** 명시.
- null률 > 30% → ⚠️ + "NT-2b 진단" 포인터.
- baseline: 어제 같은 단계 대비 delta(예: 통과 3→0).
- 기존 "LLM 분석률: 1%" 한 줄은 **제거**(착시 원인). 호환 위해 남길 경우 분모 라벨을 "Tier A+ 통과(K)"로 교정.

## 검증 / 행위보존

- 본 변경은 **표시·집계 추가** → 기존 점수/분류/분석 함수는 IDENTICAL(무변경) 확인.
- 회귀: 보고서 생성 테스트 있으면 통과. 없으면 **알려진 날짜로 수동 대조 — 어제(6/3) 데이터에서 K=3, J=3 재현**되는지.
- before/after: 같은 raw 데이터에서 옛 "1%"의 분자·분모 재료와 새 퍼널의 N·J가 일치하는지(같은 소스에서 파생됨) 확인.
- 본 지표는 결정적 검증(고점수 글 강제 트리거)과 **독립** — J/K가 100% 아니면 퍼널이 그걸 드러내므로, 검증을 기다리지 않고 진행 가능(오히려 검증 보조).
