## §1. Phase 1 — 세션 마커 + 자동화 존중 (최우선, 2차 사고 계열 봉쇄)

1. **마커 규격**: 트리 루트에 `.session-marker`(JSON: session_id·track·created_at·purpose). 생성·갱신·제거는 §3 헬퍼가 담당하되 수동 생성도 유효.
2. **자동화 측 존중 로직** (0-1에서 특정한 스크립트에 삽입):
   - 리셋·rehome 대상 트리에 마커 존재 + `created_at` 24h 이내 → **skip + 로그 1줄**("marker respected: <트리> <track>").
   - 마커 24h 초과(**TTL — 고아 마커의 영구 봉쇄 방지**) → "stale marker" 경고 로그 + 리셋 진행 + 마커 제거. 자기 치유 경로.
   - 마커 부재 → 현행 동작.
3. **워커 트리 예외 명시**: sv-worker-runtime은 마커 대상이 아님 — R1(자동화 단독 소유)대로 세션이 마커를 놓는 것 자체가 위반. 헬퍼가 워커 트리 경로에 마커 생성을 거부하게.
4. 테스트: 신선 마커 skip / stale 마커 치유 / 마커 부재 현행 / 워커 트리 마커 거부 — 자동화 스크립트에 dry-run 모드가 없으면 추가(테스트 가능성 확보).

## §2. Phase 2 — 공유 트리 가드 (3차 사고 계열)

git hook의 기술 한계를 존중해 설계(체크아웃을 사전 차단하는 native hook은 없음 — 차단은 가능한 지점에서, 불가한 지점은 경고+기록):

1. **post-checkout 경고**: 공유 트리에서 보호 브랜치(main 등 0-3에서 확정) 체크아웃 시 즉시 경고 출력 + 로그("shared-tree protected checkout").
2. **pre-push 차단**: 공유 트리에서 main으로의 직접 push 거부 — 통합은 원격 전용 refspec(R4) 또는 격리 worktree 경유만. 우회 안내 메시지 포함.
3. **pre-commit 차단**: 공유 트리에서 보호 브랜치 직접 커밋 거부(트랙 브랜치는 허용 — TH-1류 정상 작업 무마찰).
4. **이탈 검사는 hook이 아닌 헬퍼로**: "dirty 상태로 세션 종료" 차단은 git이 개입할 지점이 없으므로 §3 `wt-close`·세션 종료 규약이 담당(wip 커밋 프롬프트).
5. 테스트: 각 hook의 차단/경고/허용 3분기. hook 설치는 setup 스크립트로(0-4 결과 반영) — 수동 복사 의존 금지.

## §3. Phase 1·2 부속 — 헬퍼 (Opt-1 흡수)

- `wt-open <track>`: 격리 worktree 생성(명명 규약 sv-<track>) + 마커 자동 생성 + main 기준 브랜치 생성.
- `wt-close`: dirty면 wip 커밋 프롬프트(전용 브랜치) → 마커 제거 → worktree 제거 → 원격 백업 확인.
- 공유 트리용 `session-start`/`session-end`: 마커 생성/제거 + dirty 검사.
- 문서: docs/harness/에 사용 규약 1장(신규 세션 온보딩용 — "새 실행 세션이 문서를 안 읽어도 hook·헬퍼가 막는다"가 목표지만, 읽을 문서도 둔다).

## §5. 배포·완료 기준

1. Phase별: 테스트 GREEN → main merge(P-0 산출물) → 해당 가드의 **실동작 1회 실측**(마커 skip 로그 / hook 차단 재현 / verify 신규 항목 PASS)이 Phase 완료 조건 — 코드 반영≠완료.
2. 전체 완료 시: DECISIONS 기록("동시성 방어를 규율에서 가드로 전환 — 사고 3건·Opt-2 채택 근거") + **0-5의 임시 규칙 일괄 폐지**(마커·hook이 대체) + PROGRESS 봉인.
3. 완료 후 상태 = 정리 목록 ⓒ 종결 → ⓓ SEC β 착수 가능 + D2 트랙 클로즈(§C 양식) 준비 완료.
