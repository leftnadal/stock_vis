# 실시간 시장 타임라인 (Real-time Timeline Story) 도입 계획서

## 1. 개요 (Overview)
현재 채택된 '데이터 중심(Data-Heavy)' SaaS 대시보드 구조 내에서, 추후 유저의 직관적인 시장 흐름 파악을 돕기 위해 **'시간 흐름에 따른 스토리텔링 뷰(Timeline Flow)'**를 서브 기능 또는 별도 탭으로 도입하기 위한 기획 문서입니다.

## 2. 해결하고자 하는 문제 (Why we need this)
- 정보의 파편화: 뉴스는 뉴스 탭에, EOD 시그널은 시그널 탭에, 가설(Thesis)은 가설 탭에 흩어져 있어 **"이 뉴스가 떠서 -> 이런 시그널이 발생했고 -> 내 가설이 발동되었다"**는 인과관계를 한눈에 파악하기 어렵습니다.
- 액션의 장벽: 데이터 중심 뷰는 꼼꼼한 관리에는 좋지만, 초보 투자자나 바쁜 출근길 유저에게는 정보량이 다소 무겁게 다가올 수 있습니다.

## 3. 핵심 컨셉 (Core Concept)
- **통합된 이벤트 스트림 (Unified Event Stream)**
  - 밤사이 발생한 거시경제 뉴스, 개별 종목의 EOD 시그널, 그리고 사용자가 미리 설정해둔 가설(Thesis)의 일치율 변화를 하나의 수직 타임라인에 시간 순서대로 렌더링합니다.
- **스토리텔링 기반의 UX**
  - 예시: 
    - `[04:00 AM]` 📰 파월 의장 매파적 발언 (매크로 지표 하락)
    - `[05:30 AM]` ⚡ 테슬라(TSLA) 하향 돌파 EOD 시그널 발생
    - `[06:00 AM]` 🎯 **[내 가설 통제실]** '전기차 숏 포지션' 일치율 90% 도달 ➡️ **[매도 버튼 활성화]**

## 4. UI/UX 구현 아이디어
- 타임라인 노드 색상화: 뉴스(회색), EOD 시그널(파란색), 가설/액션(빨간색/초록색)으로 시각적 위계를 분리합니다.
- 대시보드 내 통합: 우선채택된 Variation 1 (SaaS Data-heavy) 레이아웃 우측의 'Morning Pulse' 영역을 고도화하여 미니 타임라인 위젯 형태로 먼저 테스트(A/B Test)해볼 수 있습니다.

## 5. 추후 개발 단계 (Phase)
1. **데이터 파이프라인 통합**: 뉴스 API 데이터와 EOD 시그널 발생 타임스탬프를 하나의 DB 테이블(Event Log)로 조인하는 백엔드 로직 구성.
2. **미니 위젯 시도**: 대시보드 우측 하단에 좁은 타임라인 위젯을 배치하여 반응 확인.
3. **독립된 뷰 구성**: 유저 반응이 좋을 경우, Chain Sight 처럼 하나의 거대한 'Story / Timeline' 독립 페이지 탭으로 격상.
